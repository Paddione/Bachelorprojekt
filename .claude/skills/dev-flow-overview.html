Schritt −1

Stale-Worktree-Audit

Läuft immer zuerst — vor Pfad- oder Branch-Entscheidung. Findet Worktrees zu Branches, die längst in main gemergt sind, und bietet Cleanup an. Kein Blocker — wenn der User weitermachen will, einfach fortfahren.

Audit-Pass Lesend & idempotent
a
Worktrees enumerieren
git worktree list
# auch porcelain für Parsing:
git worktree list --porcelain
b
Gemergte Branches × aktive Worktrees
MERGED=$(git branch --merged main | grep -vE '^\*|main|HEAD' | tr -d ' ')
for branch in $MERGED; do
  WT=$(git worktree list --porcelain \
    | awk -v b="refs/heads/$branch" '/^worktree/{wt=$2} $0==("branch " b){print wt}')
  [[ -n "$WT" ]] && echo "⚠️  STALE: $branch → $WT (already merged into main)"
done
c
Cleanup auf Anfrage
Bei Treffern: User informieren, Cleanup anbieten. Wenn bestätigt: 
git worktree remove <path> --force
git branch -D <branch>
git push origin --delete <branch> 2>/dev/null || true
Idempotent: kein Treffer → still und weiter mit Schritt 0. Treffer ignorieren ist ok — Cleanup bleibt für später möglich.

Skill 2

dev-flow-execute

Ausführung feature/* · fix/* Startet auf dem von dev-flow-plan gepushten Branch
0
Worktree-Konsistenz prüfen Guard
Wo bin ich und auf welchem Branch? 
CURRENT_BRANCH=$(git branch --show-current)
CURRENT_DIR=$(pwd)
  • main + Haupt-Repo → falscher Ausgangspunkt; in den Feature-Worktree wechseln.
  • Branch passt zum Verzeichnis (z.B. feature/foo in .../Bachelorprojekt-feature-foo) → ✓ weiter.

5.5
PR-Link im Ticket speichern Neu
Direkt nach dem PR-Erstellen — idempotent via WHERE NOT EXISTS: 
PR_NUM=$(gh pr view --json number -q '.number')
TICKET_UUID=$(kubectl exec "$PGPOD" ... -At -c \
  "SELECT id FROM tickets.tickets WHERE external_id = '$TICKET_ID';")

kubectl exec "$PGPOD" ... -c "
  INSERT INTO tickets.ticket_links (from_id, kind, pr_number)
  SELECT '$TICKET_UUID', 'pr', $PR_NUM
  WHERE NOT EXISTS (
    SELECT 1 FROM tickets.ticket_links
    WHERE from_id = '$TICKET_UUID' AND kind = 'pr' AND pr_number = $PR_NUM
  );"
to_id bleibt NULL (kein NOT NULL-Constraint). Kein UNIQUE — daher WHERE NOT EXISTS.
6

CI muss grün sein vor dem Merge. Auto-Merge ist konfiguriert — PRs sind nur für die History.
6.5
Ticket abschließen Neu
Falls $TICKET_ID gesetzt, nach Merge:
  • Feature: resolution = 'shipped'
  • Fix: resolution = 'fixed'
UPDATE tickets.tickets
  SET status = 'done', resolution = '$RESOLUTION'
WHERE external_id = '$TICKET_ID';

INSERT INTO tickets.ticket_comments (ticket_id, body, visibility)
SELECT id,
  'PR #$PR_NUM merged. Plan archived to tickets.ticket_plans in Postgres.',
  'internal'
FROM tickets.tickets WHERE external_id = '$TICKET_ID';
Meldung: „Ticket $TICKET_ID → done ($RESOLUTION)"
7
Plan → Postgres archivieren · Datei löschen Geändert
Neuer Flow seit PR #719: Plan wird in tickets.ticket_plans (Postgres) abgelegt, die Datei wird gelöscht — nicht mehr nach docs/superpowers/plans/executed/ verschoben. 
# SQL via temp-file → keine Shell-Expansion des Plan-Inhalts
TMPFILE=$(mktemp /tmp/plan-archive-XXXXXX.sql)
{
  printf "INSERT INTO tickets.ticket_plans (ticket_id, slug, branch, content, pr_number)
VALUES ('%s','%s','%s', \$plan\$" "$TICKET_UUID" "$SLUG" "$BRANCH"
  cat "$PLAN_FILE"
  printf "\$plan\$, %s );\n" "$PR_NUM_SQL"
} > "$TMPFILE"

kubectl exec -i "$PGPOD" ... psql ... -v ON_ERROR_STOP=1 < "$TMPFILE"

rm "$TMPFILE" "$PLAN_FILE"
git add "$PLAN_FILE"
git commit -m "chore(plans): archive $SLUG → postgres [$TICKET_ID]"
git push
Dollar-Quoting $plan$...$plan$ erlaubt beliebigen Markdown. Falls kein Ticket: nur rm "$PLAN_FILE" + Commit (keine SQL-Archivierung).
7.5
Worktree & Branch bereinigen Neu
Immer ausführen nach Merge + Plan-Archivierung — verhindert die staffelnden Altlasten, die Schritt −1 bei der nächsten Session aufräumen müsste: 
BRANCH="feature/<slug>"   # oder fix/ / chore/
WORKTREE_PATH=$(git worktree list --porcelain \
  | awk -v b="refs/heads/$BRANCH" '/^worktree/{wt=$2} $0==("branch " b){print wt}')

cd /home/patrick/Bachelorprojekt    # raus aus dem Worktree

[[ -n "$WORKTREE_PATH" && "$WORKTREE_PATH" != /home/patrick/Bachelorprojekt ]] \
  && git worktree remove "$WORKTREE_PATH" --force

git branch -D "$BRANCH" 2>/dev/null || true
git push origin --delete "$BRANCH" 2>/dev/null || true  # GitHub macht das beim Auto-Merge meist schon
Ergebnis: kein staler Worktree, kein toter Branch lokal/remote.

Failure-Handling

CI rot vor Merge

Diagnose, Fix auf demselben Branch, neu pushen. Keinen zweiten PR aufmachen. Nach 2 erfolglosen Versuchen: INSERT in tickets.ticket_comments mit Branch-Verweis, Ticket bleibt auf in_progress.

Deploy scheitert post-merge

Loggen, Patrick benachrichtigen, Cluster wie ist lassen. Kein Auto-Rollback. Ticket bleibt in_progress — Patrick setzt manuell auf done oder blocked.

Verify scheitert post-merge

Neuen fix/<slug>-Branch via dev-flow-plan Fix-Pfad. Regression als Bug behandeln. Am bestehenden Ticket internen Kommentar mit Link zum neuen Ticket hinterlassen.
