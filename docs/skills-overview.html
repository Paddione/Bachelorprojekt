dev-flow-plan
Planung & Branch-Setup
Einstiegspunkt für jede Repo-Änderung. Wählt den Pfad (Feature / Fix / Chore), legt den Worktree an, führt Brainstorming durch und committed Plan + Spec auf den Branch. Chores werden vollständig inline ausgeführt.
Feature Fix Chore Brainstorming Worktree
dev-flow-execute
Plan-Ausführung & Pull Request
Startet nach dev-flow-plan auf einem feature/*- oder fix/*-Branch. Liest den committed Plan, implementiert via parallele Subagenten, verifiziert, erstellt PR und mergt.
Subagenten PR Verifikation Deploy
dev-flow-e2e
Playwright E2E-Tests
Schreibt und führt Browser-End-to-End-Tests für die gerade implementierte Funktion aus. Läuft nach dem Merge von dev-flow-execute gegen die Live-Umgebung. Nutzt die Playwright-MCP-Browsertools.
Playwright Browser Live-Env Nach Merge
☁️Infrastruktur & Deployment
deployment-assist
Interaktiver Deployment-Guide
Führt durch das Deployment auf eine frische oder teilweise aufgesetzte Umgebung. Prüft Umgebungswahl, Credentials, aktuellen Cluster-Status und führt Tasks phasenweise aus.
Onboarding Phased Deploy Credentials
new-environment
Neue Umgebung aufsetzen
Setzt einen neuen Cluster von Grund auf auf. Erzwingt die Pflicht-Reihenfolge Sealed Secrets → cert-manager → workspace:deploy, um das stille Credential-Überschreib-Problem zu verhindern.
Cluster-Init Reihenfolge SealedSecrets
hetzner-node
Hetzner-Node Provisioning
Interaktives Runbook für neue oder zurückgesetzte Hetzner-Server. Verwaltet SSH-Keys, generiert cloud-config, führt Rescue-Mode-Reinstall durch und verbindet den Node mit WireGuard-Mesh + k3s-Cluster.
Hetzner WireGuard k3s Join Cloud-Init
livekit-setup
LiveKit WebRTC Setup & Repair
Setup und Reparatur des LiveKit-Streaming-Stacks auf mentolder. Deckt DNS-Pinning, ufw-Firewall-Regeln, Node-Affinity, ICE-Fehlerdiagnose sowie Stream- und Recording-Verifikation ab.
LiveKit WebRTC DNS-Pin ICE mentolder only

Schema-Migration (beide Cluster)
Führt durch das sichere Schreiben von Migrations-SQL, Anwendung auf beide Produktionscluster, Re-Grant von Berechtigungen, Schema-Verifikation und Aktualisierung des ER-Diagramms.
PostgreSQL Beide Cluster Permissions ER-Diagramm
backup-check
Backup-Audit & Restore-Test
End-to-End-Audit des Backup-Systems: Setup prüfen → Live-Backup auslösen → verschlüsselte Dateien verifizieren → sicheren Restore in eine Temp-DB durchführen → gefundene Probleme beheben.
Backup Restore Verschlüsselung Audit
knowledge-reindex
Knowledge Re-Indexing
Re-indiziert Knowledge-Collections nach Änderungen. Behandelt SOURCE-Optionen, Embedding-Model-Auswahl und LLM-Verfügbarkeit. Kritisch: bge-m3- und voyage-Collections schlagen geschlossen fehl — niemals mischen.
pgvector bge-m3 Voyage Embeddings

secret-rotation
Secret-Rotation (beide Cluster)
Sichere, geordnete Rotation von DB-Passwörtern, API-Keys, SealedSecrets-Keypairs nach Cluster-Resets und Claude-Code-Tokens. Erzwingt die Reihenfolge, die stilles Überschreiben von Produktions-Secrets verhindert.
SealedSecrets DB-Passwörter API-Keys Cluster-Reset
keycloak-realm-sync
Keycloak Realm-Synchronisation
Reconciliert die Keycloak-Realm-Konfiguration aus den eingecheckten Realm-JSON-Dateien. Deckt OIDC-Client-Settings, Group-Mappings, Mapper-Konfiguration und SSO-Login-Fehler durch Realm-Drift ab.
Keycloak OIDC SSO Realm-JSON

Produktionsvorfall-Reaktion

Triage Rollback Post-Mortem Production

Ticket-Triage & Auflösung
Arbeitet offene Tickets in der mentolder-Datenbank durch. Kategorisiert nach Bearbeitbarkeit, behebt ai_ready-Tickets autonom und routet needs_human-Tickets mit klarer Anfrage weiter.
Tickets DB ai_ready Triage Auto-Fix

Coaching-Content Pipeline
End-to-End-Workflow für Coaching-Bücher/PDFs: Ingest → Klassifizierung via LLM → Review unter /admin/knowledge/drafts → Live-Wissensbasis. Deckt lokale LLM-Anforderungen und Fehlerdiagnose ab.
PDF-Ingest Klassifizierung LLM Knowledge

Mishap-Tracking ist in jedem Skill aktiv. Jede Anomalie, jeder Drift und jede Sicherheitsauffälligkeit — auch außerhalb der aktuellen Aufgabe — wird im MISHAP_LOG gesammelt. Am Ende ruft jeder Skill automatisch mishap-tracker auf.
mishap-tracker
Mishap-Log → Tickets
Konvertiert MISHAP_LOG-Einträge aus dem aufrufenden Skill in tickets.tickets-Datensätze auf dem mentolder-Cluster. Wird niemals direkt aufgerufen — immer aus dem Post-Execution-Block eines anderen Skills.
Passiv Immer aktiv Tickets DB
