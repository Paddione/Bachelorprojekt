Arena & Brett Deployment

Build, Deploy und Debugging des Arena-Servers (korczewski-only WebSocket) und des 3D-Spiels Brett — inklusive Proto-Drift-Check, CORS-Konfiguration und cluster-spezifischen Constraints.

Auslöser: Änderungen an arena-server/, brett/ oder Deployment-Probleme bei diesen Services

Arena Server (korczewski-only)

ℹ️
task arena:deploy ENV=mentolder bricht mit einer Erklärung ab — Arena läuft intentionell nur auf korczewski. 
task arena:build                   # Image bauen (+ k3d import in dev)
task arena:push                    # Push zu ghcr.io/paddione/arena-server:latest
task arena:deploy ENV=korczewski   # Build, push und rollout
task feature:arena                 # Kurzform: build + deploy zu korczewski

Proto-Drift Copy Step (CI-erzwungen)

arena-server/src/proto/messages.ts und website/src/components/arena/shared/lobbyTypes.ts müssen byte-identisch sein. CI schlägt fehl wenn sie divergieren.

# Nach jeder Änderung an messages.ts
cp arena-server/src/proto/messages.ts website/src/components/arena/shared/lobbyTypes.ts
diff arena-server/src/proto/messages.ts website/src/components/arena/shared/lobbyTypes.ts
# Muss keine Ausgabe produzieren

CORS-Konfiguration

Arena validiert JWTs aus beiden Keycloak-Realms und erlaubt CORS von beiden Website-Origins:

  • https://web.mentolder.de
  • https://web.korczewski.de

Beide Websites zeigen auf arena-ws.korczewski.de. Bei CORS-Änderungen beide Origins in der Allow-List lassen.

Arena Day-2 Kommandos

task arena:status ENV=korczewski   # Pod + Service-Status
task arena:logs ENV=korczewski     # Logs tailen
task arena:db ENV=korczewski       # psql in arena-Schema
task arena:sync ENV=korczewski     # Hot-Copy src in laufenden Pod + rebuild (kein image push)
task arena:teardown ENV=<env>      # Ressourcen entfernen

Arena Unit Tests (vor PR erforderlich)

cd arena-server && pnpm install --frozen-lockfile && pnpm test && pnpm build

Brett (3D Multiplayer-Spiel)

task brett:build               # Image bauen (+ k3d import in dev)
task brett:push                # Push zur Registry
task brett:deploy ENV=<env>    # Build, import/push und rollout
task feature:brett             # Fan-Out: build + deploy auf BEIDE Cluster
task brett:logs ENV=<env>      # Logs tailen

npm ci --prefix brett && \

       brett/test/damage.test.mjs \

# Systembrett-Template-Validierung
./scripts/tests/systembrett-template.test.sh

Brett-spezifische Hinweise

  • Brett-Image nutzt :latest Tag absichtlich — nicht auf einen Digest pinnen.
  • Das Mayhem-Admin-Panel im Spiel nutzt OIDC (Keycloak) — Realm-Config-Änderungen können das In-Game-Admin brechen.
  • task brett:bot-setup ENV=<env> registriert den /brett Slash-Command in Nextcloud Talk — nach Erstdeploy oder Talk-Integration-Änderungen ausführen.

Verifikation nach Deploy

# Arena
task arena:status ENV=korczewski
# wss://arena-ws.korczewski.de prüfen + JWT von beiden Realms validieren

# Brett
task brett:logs ENV=mentolder
task brett:logs ENV=korczewski
# brett.mentolder.de und brett.korczewski.de im Browser prüfen
