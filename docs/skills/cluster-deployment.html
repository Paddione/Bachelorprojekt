Cluster Initialization & Deployment

Deploying new environments from scratch, diagnosing cluster health, performing gap analysis, managing Flux GitOps reconciliations, and running dev.mentolder.de stack operations.

Auslöser: Environment initialization, cluster degraded state, manual Flux sync, or local dev stack tasks.
⚠️
Zwingende Reihenfolge: SealedSecrets must exist before applying SealedSecrets, and cert-manager CRDs must be present before running the main deployment.
1 Environment Prep Scaffold Config & Secrets

Initialize environmental configs and validate against schema. Seal plaintext files using the retrieved certificate:

task env:init ENV=<env>
task env:seal ENV=<env>
2 Deploy & Bootstrap Core & Flux Setup

Apply cert-manager resources, build base manifests, and bootstrap Flux GitOps for automated reconciliation:

task workspace:deploy ENV=<env>
flux reconcile source git flux-system --context <ctx>
3 Gap Analysis & Repair Cluster Assistance

Inspect pod states, identify missing credentials or degraded deployments, and trigger appropriate task flows (OIDC sync, office suite setup, etc.) to resolve gaps.

4 dev.mentolder.de Stack Local Dev Box Operations

Manage dev k3d clusters hosted on k3s-1. Apply unsealed secrets and trigger DB schema refreshes from prod snapshots:

task dev:_materialise-secrets
task dev:db:refresh

Troubleshooting

SymptomLösung
SealedSecret rejectedDelete plain secret first: kubectl delete secret knowledge-secrets -n <ns>.
403 on dev.mentolder.deVerify user is mapped to Keycloak /dev-access group.
Flux out of syncForce-sync git repository source first, then reconcile the workspace kustomization.
