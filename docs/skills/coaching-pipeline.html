Coaching-Content Pipeline

Verarbeitet Coaching-Inhalte von der Quelldatei bis zur Verfügbarkeit im Coaching-Assistenten. Ingest → UNCLASSIFIED Chunks → Classify → DRAFT → Review → PUBLISHED. Nur auf mentolder.

Auslöser: Neuen Coaching-Inhalt (PDF/EPUB) indizieren und klassifizieren
Pipeline-Übersicht
Quelldatei → coaching:ingest → UNCLASSIFIED Chunks → coaching:classify → DRAFT → Review → PUBLISHED
  • ClusterNur mentolder
  • Review-UIhttps://web.mentolder.de/admin/knowledge/drafts
  • Coaching-Portalhttps://web.mentolder.de/portal/coaching
  • bge-m3 EmbeddingsGPU-Host TEI:8081 — fail-closed wenn down
Phasen
1 Ingest PDF oder EPUB in pgvector laden 
task coaching:ingest -- <datei.pdf> <slug> [--title="Buchtitel"] [--author="Autorenname"]

# Beispiele:
task coaching:ingest -- /tmp/mein-buch.pdf systemische-aufstellung \
  --title="Systemische Aufstellung" --author="Max Mustermann"

task coaching:ingest -- /tmp/coaching.epub coaching-grundlagen

Nach Ingest: Chunks mit Status UNCLASSIFIED in der DB. Embeddings werden während Ingest erstellt (bge-m3 oder voyage je nach Collection-Config).

2 Classify UNCLASSIFIED → DRAFT via LLM 
# Einzelnes Buch klassifizieren
task coaching:classify -- --slug=<slug>

# Alle UNCLASSIFIED klassifizieren
task coaching:classify -- --all

LLM-Pfade (Priorität):

PfadBeschreibungVoraussetzung
A — Cluster llm-router Bevorzugt — GPU-Host auf wg-mesh GPU-Host erreichbar, LLM_ENABLED=true
B — Anthropic API Fallback wenn GPU-Host down Anthropic API-Key gültig (prüfen!)
C — WSL Ollama Workaround wenn A + B nicht verfügbar Lokaler Docker LiteLLM, direkt zu Ollama
⚠
Anthropic-Key auf mentolder kann ungültig sein (401-Fehler). Bei B-Pfad-Problemen: key rotieren via secret-rotation Typ E.
3 Drafts reviewen Im Admin-UI oder per SQL

Browser UI:

# URL
https://web.mentolder.de/admin/knowledge/drafts

Aktionen im UI: Approve (→ PUBLISHED), Reject (→ gelöscht), Edit (Text bearbeiten)

Bulk-Approve via SQL:

-- Alle Drafts eines Buches approven
UPDATE coaching.chunks
SET status = 'PUBLISHED', updated_at = now()
WHERE book_slug = '<slug>'
  AND status = 'DRAFT';
4 Verfügbarkeit verifizieren SQL + Live-Test 
-- Status-Count prüfen
SELECT status, COUNT(*) FROM coaching.chunks
WHERE book_slug = '<slug>'
GROUP BY status;

-- Erwartetes Ergebnis: PUBLISHED = N, kein UNCLASSIFIED

Live-Test: https://web.mentolder.de/portal/coaching → Frage zum neuen Inhalt stellen → Antwort mit Quellenangabe erhalten.

Häufige Probleme
SymptomUrsacheLösung
classify hängt / keine Antwort GPU-Host nicht erreichbar (wg-mesh) GPU-Host prüfen; zu Anthropic-Fallback wechseln
401 bei classify Anthropic API-Key ungültig auf mentolder secret-rotation Typ E für ANTHROPIC_API_KEY
MixedEmbeddingModelError Cross-Collection-Query bge-m3 + voyage Collections trennen; NIEMALS modell-übergreifend abfragen
TEI 503 GPU-Host down oder TEI-Container gestoppt GPU-Host neu starten; voyage-Collections unberührt
Chunks bleiben UNCLASSIFIED coaching:classify nicht ausgeführt task coaching:classify -- --slug=<slug>
Ingest fehlgeschlagen Dateiformat nicht unterstützt oder Encoding-Fehler PDF/EPUB validieren; ggf. re-exportieren
ℹ
Embedding-Isolation: bge-m3 verwendet TEI auf dem GPU-Host (Port 8081). voyage-multilingual-2 verwendet Voyage AI direkt. Diese beiden Vektor-Räume sind inkompatibel — nie in derselben Query mischen.
