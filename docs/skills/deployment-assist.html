Infrastruktur deployment-assist

Interaktiver Deployment-Guide

Führt durch das Deployment in einer neuen oder teilweise aufgesetzten Umgebung. Erkennt automatisch den aktuellen Zustand, identifiziert Lücken und erstellt einen geordneten Ausführungsplan.

Auslöser: Frischer Clone, teilweise aufgesetzte Umgebung, oder Diagnose-Anfrage

1 Prerequisite Tools prüfen Pflicht-Tools verifizieren 
# Alle benötigten Tools prüfen
docker --version
kubectl version --client
task --version
k3d version
kubeseal --version

# kubectl-Kontext prüfen
kubectl config current-context
kubectl config get-contexts
2 Credentials & Config-Assessment Secrets-Status ermitteln
  • environments/<env>.yaml — Existiert? Vollständig?
  • environments/.secrets/<env>.yaml — Plaintext-Secrets vorhanden? (gitignored)
  • environments/sealed-secrets/<env>.yaml — Committed SealedSecret?
  • environments/certs/<env>.pem — Sealing-Cert aktuell?
task env:validate ENV=<env>
task env:show ENV=<env>
3 Cluster & Namespace-Status Aktueller Zustand 
kubectl get nodes
kubectl get pods -n workspace
kubectl get pvc -n workspace
kubectl get ingress -n workspace
task workspace:status ENV=<env>

5 Ausführungsplan Geordnete Schrittliste

Dev-Sequenz (Schnellstart):

task cluster:create
task workspace:up    # Cluster → MVP → Office → MCP → post-config

Prod-Sequenz (15 Schritte):

  1. task sealed-secrets:install ENV=<env>
  2. task env:fetch-cert ENV=<env>
  3. task env:generate ENV=<env> (MANAGED_EXTERNALLY ersetzen)
  4. task env:seal ENV=<env> + commit
  5. task cert:install ENV=<env>
  6. task cert:secret -- <ipv64-key> ENV=<env>
  7. task workspace:deploy ENV=<env>
  8. task workspace:office:deploy ENV=<env>
  9. task workspace:post-setup ENV=<env>
  10. task workspace:talk-setup ENV=<env>
  11. task workspace:recording-setup ENV=<env>
  12. task workspace:whiteboard-setup ENV=<env>
  13. task workspace:admin-users-setup ENV=<env>
  14. task workspace:theme ENV=<env>
  15. task workspace:verify ENV=<env>
6 Execute & Monitor Schritt für Schritt 
# Pod-Status überwachen
kubectl get pods -n workspace -w

# Logs bei Fehler
task workspace:logs ENV=<env> -- <service>

# Finaler Status-Check
task workspace:status ENV=<env>
task workspace:verify ENV=<env>
