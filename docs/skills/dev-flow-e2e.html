Dev-Flow dev-flow-e2e

Playwright E2E Tests

Schreibt Playwright-Testspecs nach einem gemergten und deploymenten Feature. Erkundet die Live-Umgebung via Browser-Tools, schreibt aussagekräftige Tests und registriert sie in der Testsuite.

Auslöser: Nach dev-flow-execute gemergt und deployt — neue Funktionalität braucht E2E-Abdeckung
  • Vorgänger-Skilldev-flow-execute — nach Merge & Deploy
  • Test-RunnerPlaywright — tests/e2e/
  • Configplaywright.config.ts
  • Inventorywebsite/src/data/test-inventory.json
  • Prod-URLweb.mentolder.de (Primär)
⚠
Test-Inventory-Check ist CI-blocking. Nach jedem Test-File-Add muss task test:inventory lokal laufen und das generierte test-inventory.json mitgecomimt werden. CI schlägt fehl wenn die Datei abweicht.

Den letzten gemergten PR + geänderte Dateien analysieren. Bestimmen was getestet werden soll.

gh pr list --state merged --limit 5

3 Test-Spec schreiben Naming-Konventionen einhalten

Datei-Naming:

  • Neues Feature: fa-<NN>-<slug>.spec.ts
  • Bug-Regression: fa-bug-t<TICKETID>.spec.ts

Test-Template-Struktur:

import { test, expect } from '@playwright/test';

// Gruppe 1: Auth-gating
test.describe('Feature Auth-Gating', () => {
  test('unauthenticated user sees login redirect', async ({ page }) => {
    await page.goto('/feature-path');
    await expect(page).toHaveURL(/auth\./);
  });
});

// Gruppe 2: Funktionale Tests (nach Login)
test.describe('Feature Funktionalität', () => {
  test.beforeEach(async ({ page }) => {
    // Auth-Setup
  });

  test('happy path', async ({ page }) => {
    // Hauptfunktionalität testen
  });
});

// Gruppe 3: Negative Tests
test.describe('Feature Edge Cases', () => {
  test('invalid input shows error', async ({ page }) => {
    // Fehlerfälle
  });
});
ℹ
Niemals Tests durch Ändern von Assertions "fixen". Ein roter Test zeigt einen echten Fehler an.
4 Spec in playwright.config.ts registrieren Richtige Projektgruppe 
// playwright.config.ts
{
  name: 'website',
  testMatch: '**/fa-*.spec.ts',
  use: {
    baseURL: process.env.WEBSITE_URL ?? 'https://web.mentolder.de',
  },
}
5 Tests ausführen Mit WEBSITE_URL env var 
# Gegen Produktion (mentolder)
WEBSITE_URL=https://web.mentolder.de npx playwright test fa-<NN>-<slug>.spec.ts

# Gegen korczewski
WEBSITE_URL=https://web.korczewski.de npx playwright test --project=korczewski

# Alle Website-Tests
npx playwright test --project=website

# Mit UI (lokal)
npx playwright test --ui
6 test-inventory.json aktualisieren CI-blocking — nicht vergessen 
task test:inventory

Das generierte website/src/data/test-inventory.json committen. CI schlägt fehl wenn die Datei abweicht.

7 Commit & Push Test-Spec + Inventory zusammen 
git add tests/e2e/fa-<NN>-<slug>.spec.ts website/src/data/test-inventory.json
git commit -m "test(e2e): add FA-<NN> spec for <feature>"
git push origin main
8 Full Smoke Run (optional) Alle Tests gegen Prod 
# Nightly via GitHub Actions e2e.yml
# Manuell:
./tests/runner.sh local
./tests/runner.sh report
