Dev Flow Execute

Umsetzung von Implementierungsplänen: Code-Schreiben, Verifizierung, Pull Request Management und finaler Release auf Produktion.

Auslöser: Existierender Plan in docs/superpowers/plans/ auf einem Feature-Branch
1 Sync & Setup Git & Ticket Status

Synchronisiere den Branch mit main und setze das Ticket auf in_progress.

git fetch origin main
git rebase origin/main
task tickets:status ID=<id> NEXT=in_progress
2 Implementierung Subagent Driven Development

Arbeite den Plan Schritt für Schritt ab. Nutze spezialisierte Subagents für Backend, UI oder Infrastruktur.

ℹ️
Worktree-Pflicht: Alle Dateioperationen müssen innerhalb des Feature-Worktrees stattfinden, niemals direkt im Haupt-Repo.
3 Qualitätssicherung Tests & Admin-Gate

Validiere die Änderungen lokal:

task test:all
bash scripts/admin-menu-gate.sh
4 Release PR, Merge & Deploy

Erstelle den PR, merge nach CI-Success und stoße das Deployment an:

gh pr create --title "feat(...): ..."
gh pr merge --squash --delete-branch
task feature:website # oder feature:deploy
