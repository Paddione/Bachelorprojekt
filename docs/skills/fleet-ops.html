Fleet Operations

Gleichzeitiger Betrieb beider Prod-Cluster (mentolder + korczewski): Fan-Out-Deploys, Cross-Cluster Schema-Änderungen und Cluster-Health-Checks.

Auslöser: Deploy, Schema-Änderung oder Operation die beide Cluster betrifft
⚠️
Zwei unabhängige Cluster: mentolder und korczewski teilen keinen Storage, keine DB und keinen Sealed-Secrets-Controller. Jede Shared-State-Änderung muss explizit auf beiden angewendet werden.
ClusterContextNamespaceDomain
mentoldermentolderworkspaceweb.mentolder.de
korczewskikorczewskiworkspace-korczewskiweb.korczewski.de

Fan-Out Deploy-Kommandos

task feature:deploy         # workspace:deploy + post-setup auf BEIDEN Clustern
task feature:website        # Astro website rebuild + rollout auf BEIDEN Clustern
task feature:brett          # Brett rebuild + rollout auf BEIDEN Clustern
task feature:livekit        # LiveKit DNS-Pin auf BEIDEN Clustern
task health                  # Cross-Cluster Status + Konnektivitäts-Check
task workspace:verify:all-prods  # Smoke-Probes auf BEIDEN Clustern
task clusters:status         # One-Line Status über beide

Cross-Cluster Schema- / DB-Änderungen

Jeder Cluster hat seine eigene shared-db. Schema-Änderungen müssen auf beide angewendet werden:

# Apply to mentolder
task workspace:psql ENV=mentolder -- website
# Run SQL here

# Apply to korczewski
task workspace:psql ENV=korczewski -- website
# Run SQL here

SealedSecrets — Cluster-unabhängig

Jeder Cluster hat seinen eigenen Controller mit eigenem Keypair. Ein für mentolder versiegeltes Secret kann von korczewski nicht entschlüsselt werden.

# Cluster-spezifisches Sealing-Cert holen
task env:fetch-cert ENV=mentolder
task env:fetch-cert ENV=korczewski

# Dann mit dem richtigen Cert versiegeln
task env:seal ENV=mentolder
task env:seal ENV=korczewski

Keycloak Realm — pro Cluster

Jeder Cluster hat seinen eigenen Realm. OIDC-Client-Änderungen müssen in beiden gemacht werden:

task keycloak:sync ENV=mentolder
task keycloak:sync ENV=korczewski

Korczewski-spezifische Constraints

  • Arena läuft nur auf korczewski — task arena:deploy ENV=mentolder bricht mit Erklärung ab.
  • Website-Namespace ist website-korczewski, nicht website.
  • SSH-Zugang: patrick@pk-hetzner-4/6/8 (AllowUsers auf patrick gesperrt).
  • Nach erneutem Versiegeln: task workspace:sync-db-passwords ENV=korczewski ausführen.
