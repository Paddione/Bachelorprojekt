Flux GitOps Day-2 Operations

Erzwungene Reconciliation nach PR-Merge, Debugging von Drift und Flux-Suspension — auf beiden Prod-Clustern unabhängig voneinander.

Auslöser: PR gemerged, Manifeste driften, Flux suspendiert oder ImageUpdateAutomation hängt
⚠️
Immer zuerst die Source reconcilen. flux reconcile kustomization wendet die zuletzt geholte Revision an — nicht den aktuellen main. Ohne Source-Prime bekommt der Cluster den alten Commit.
1 Source primen GitRepository fetchen 
flux reconcile source git flux-system --context mentolder
flux reconcile source git flux-system --context korczewski
2 Kustomization reconcilen Manifeste anwenden 
flux reconcile kustomization workspace --context mentolder
flux reconcile kustomization workspace-korczewski --context korczewski

Flux-Status prüfen

# Gesamtstatus
flux get all --context mentolder
flux get all --context korczewski

# Nur Kustomizations (häufigster Drift-Punkt)
flux get kustomizations --context mentolder
flux get kustomizations --context korczewski

# Source-Status (sieht Flux den aktuellen main?)
flux get sources git --context mentolder

Suspend / Resume

Suspension vor manuellen Notfall-Änderungen verhindert, dass Flux sie sofort zurücksetzt:

# Suspendieren
flux suspend kustomization workspace --context mentolder

# Manuelle kubectl-Änderungen hier...

# Wieder aktivieren — Flux übernimmt wieder
flux resume kustomization workspace --context mentolder
⚠️
Nicht vergessen zu resumeн. Eine suspendierte Kustomization verfolgt main stillschweigend nicht mehr.

$$-Escaping in substituteFrom

Flux behandelt ${VAR} in YAML-Werten als Variablenreferenz. Um ein literales $ zu emittieren, $$ verwenden:

# ❌ substituteFrom ersetzt ${PROD_DOMAIN}
value: "https://${PROD_DOMAIN}/path"

# ✅ Literales $ im gerenderten Manifest
value: "https://$${PROD_DOMAIN}/path"

ImageUpdateAutomation Debug

Nicht für website, brett und docs (die nutzen task feature:* mit :latest):

flux get image update --context mentolder
flux reconcile image repository <name> --context mentolder
flux reconcile image update <name> --context mentolder
flux logs --kind=ImageUpdateAutomation --context mentolder
