Incident Response

Strukturierte Fehlerbehebung in Produktion: Triage, Ticket-Erstellung, Root-Cause-Analyse und koordinierte Wiederherstellung des Dienstes.

Auslöser: Produktions-Ausfall, 500er Fehler, Health-Check Failure oder User-Reports
1 Triage & Ticket < 2 min

Ermittle Scope, Cluster und betroffene Nutzer. Lege sofort ein Incident-Ticket an:

task tickets:create --title "Incident: <desc>" --type bug --severity critical
2 Diagnose Status, Logs & Events

Prüfe Pod-Status und Error-Signaturen in den Logs:

task workspace:status ENV=mentolder
task workspace:logs ENV=mentolder -- <service>
kubectl get events -n workspace --context mentolder --sort-by='.lastTimestamp'
3 Mitigation Fix vs. Rollback

Entscheide basierend auf der Ursache:

  • Rollback: Falls durch letzten Deploy ausgelöst und kein DB-Schema-Change involviert.
  • Fix in Place: Bei Config-Drift oder Secret-Problemen.
# Rollback Website auf vorherigen SHA
kubectl set image deployment/website website=ghcr.io/paddione/workspace-website:<SHA>
4 Abschluss Post-Mortem

Verifiziere die Lösung und hinterlasse eine Post-Mortem Note im Ticket:

UPDATE tickets.tickets SET status = 'done', notes = 'Root cause: ...';
