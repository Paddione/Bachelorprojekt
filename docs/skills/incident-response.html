Betrieb incident-response

Produktionsvorfall-Reaktion

Strukturierter Umgang mit Produktionsvorfällen: Scope eingrenzen, Ticket erstellen, diagnostizieren, fixen oder rollbacken, verifizieren und Post-Mortem notieren. Ziel: Wiederherstellung in < 30 Minuten.

Auslöser: "X is down", Pod crasht, 500-Fehler, Login nicht möglich, Health-Check schlägt fehl
Schweregrade
critical — Dienst komplett ausgefallen major — Stark beeinträchtigt minor — Teilweise beeinträchtigt
Phasen
1 Scope ermitteln (< 2 Minuten) Was ist down, seit wann, wie viele Nutzer betroffen? 
# Schnell-Überblick
task clusters:status
task workspace:status ENV=mentolder
task workspace:status ENV=korczewski

# Health-Check
task health

Klären:

  • Welche Services betroffen? (Keycloak, Nextcloud, Website, ...)
  • Welche Cluster? (mentolder / korczewski / beide)
  • Seit wann? (git log, kubectl events)
  • Blast Radius? (Login für alle, nur Datei-Upload, nur Talk, ...)
2 Incident-Ticket erstellen In mentolder Tickets-DB 
INSERT INTO tickets.tickets (

3 Diagnose Pod-Status, Logs, Events, letzte Deploys 
# Pod-Status
kubectl get pods -n workspace
kubectl describe pod <pod-name> -n workspace

# Logs
kubectl logs <pod-name> -n workspace --previous
task workspace:logs ENV=<env> -- <service>

# Events
kubectl get events -n workspace --sort-by=.lastTimestamp | tail -20

# Letzte Deploys
git log --oneline -10
kubectl rollout history deployment/<service> -n workspace

4 Fix oder Rollback Rollback wenn: letztes Deploy + keine DB-Migration

Rollback (schnell):

kubectl rollout undo deployment/<service> -n workspace
kubectl rollout status deployment/<service> -n workspace

Config-Drift-Fix:

task workspace:deploy ENV=<env>   # Re-apply Manifeste

Keycloak Realm-Drift:

task keycloak:sync ENV=<env>
bash scripts/keycloak-ensure-mappers.sh <env>

DB-Passwort-Drift:

task workspace:sync-db-passwords ENV=<env>
5 Verifikation 2 Minuten Log-Beobachtung nach Fix 
task workspace:status ENV=<env>
task workspace:verify ENV=<env>

# SSO-Login testen (Incognito)
# Logs 2 Minuten beobachten
kubectl logs -n workspace deploy/<service> -f --since=2m
6 Ticket schließen + Post-Mortem 
UPDATE tickets.tickets
SET
  status = 'done',
  resolution = 'fixed',
  comment = 'Was kaputt: <ursache>
Warum: <root cause>
Wie behoben: <fix>
Prävention: <was in Zukunft tun>',
  updated_at = now()
WHERE id = 'T######';
