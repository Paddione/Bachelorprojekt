Keycloak Realm Sync

Reconcile der Keycloak Realm-Konfiguration: OIDC Clients, Group Mappings, Attribute Mapper und Login-Flows basierend auf den JSON-Definitionen.

Auslöser: SSO-Fehler, neue OIDC Clients, Drift zwischen UI und Code oder Deployment-Verify
🚫
Source of Truth: Bearbeite Realm-Einstellungen niemals nur im UI. Änderungen müssen immer in die realm-*.json Dateien zurückfließen, sonst werden sie beim nächsten Sync überschrieben.

1 Pre-Sync Check Pod Status & Logs

Stelle sicher, dass Keycloak läuft und die Admin-Schnittstelle bereit ist:

task workspace:status ENV=<env>
task workspace:logs ENV=<env> -- keycloak
2 Sync ausführen scripts/keycloak-sync.sh

Starte den Abgleich für das gewünschte Environment:

task keycloak:sync ENV=mentolder

Das Script nutzt die Admin-API, um Clients, Scopes und Rollen zu aktualisieren.

3 Protocol Mapper ensure Custom Claims

Stelle sicher, dass spezialisierte Mapper (z.B. für JWT Claims) korrekt gesetzt sind:

bash scripts/keycloak-ensure-mappers.sh <env>
4 SSO Test End-to-End Verifizierung

Testen des Login-Flows für die Website:

  • Öffne https://web.<domain> im Inkognito-Modus.
  • Prüfe Redirect auf auth.<domain>.
  • Login mit Test-User durchführen.
  • Verify Redirect zurück zur Website mit gültiger Session.
