Knowledge Reindex

Aktualisierung der Wissensdatenbank: Neu-Indexierung von PRs, Dokumentation und Tickets unter Berücksichtigung der Embedding-Modell-Isolation.

Auslöser: Neue Merges, Dokumentations-Updates oder Ticket-Datenbank-Änderungen
🚫
Modell-Isolation: bge-m3 (TEI) und voyage-multilingual-2 sind nicht kompatibel. Mische niemals Vektoren aus verschiedenen Modellen in einer Query (MixedEmbeddingModelError).

1 Pre-Checks Modell-Verfügbarkeit prüfen

Prüfe vorhandene Kollektionen und die Erreichbarkeit der Embedding-Services:

task workspace:psql ENV=mentolder -- website <<'SQL'
SELECT collection_name, embedding_model FROM knowledge.collections;
SQL
2 Indexierung ausführen task knowledge:reindex

Starte den Reindex für eine spezifische Quelle oder alles:

task knowledge:reindex ENV=mentolder SOURCE=prs
task knowledge:reindex ENV=mentolder SOURCE=markdown
task knowledge:reindex ENV=mentolder SOURCE=all
3 Verifizierung Document Counts

Prüfe, ob die Anzahl der Dokumente gestiegen ist:

SELECT collection_name, document_count, updated_at FROM knowledge.collections;

Web-Crawler (Ad-hoc)

Web-Quellen werden separat über ihre Collection-UUID gecrawlt:

task knowledge:crawl ENV=mentolder COLLECTION_ID=<uuid>
