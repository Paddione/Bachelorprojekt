Knowledge Re-Indexing

Indiziert Knowledge-Collections neu — PRs, Markdown, Bugs und alle Quellen. Kritisch: bge-m3 und voyage-multilingual-2 Vektoren dürfen niemals gemischt werden. Jede Collection schlägt bei falschem Modell hart fehl (fail-closed).

Auslöser: Knowledge-Daten veraltet, neue Inhalte indizieren, nach Schema-Änderungen an knowledge-Tabellen
⚠
Embedding-Isolation ist nicht verhandelbar. bge-m3 und voyage-multilingual-2 Vektoren NIEMALS mischen. Eine bge-m3-Collection fragt immer mit bge-m3 und schlägt hart fehl wenn TEI down ist. Eine voyage-Collection fragt immer mit Voyage. MixedEmbeddingModelError bei Cross-Model-Queries. Keinen stillen Fallback hinzufügen — Retrieval wäre wertlos.
  • bge-m3GPU-Host wg-mesh → TEI auf Port 8081; fail-closed wenn down
  • voyage-multilingual-2Voyage AI direkt; unabhängig vom GPU-Host
  • GPU-HostRTX 5070 Ti, 16 GB — einzige Box im wg-mesh
  • Model-Swap-Latenz3–6s beim ersten Call nach OLLAMA_KEEP_ALIVE=5m Eviction
Befehle
# Alle Quellen re-indizieren
task knowledge:reindex ENV=<env>

# Nur PR-Quelle
task knowledge:reindex ENV=<env> SOURCE=prs

# Nur Markdown
task knowledge:reindex ENV=<env> SOURCE=markdown

# Nur Bugs
task knowledge:reindex ENV=<env> SOURCE=bugs

# Explizit alle
task knowledge:reindex ENV=<env> SOURCE=all
Pre-Checks
1 LLM_ENABLED und GPU-Host prüfen Vor dem Start 
# GPU-Host erreichbar?
curl -s http://llm-gateway-embed.workspace:8081/health

# Alternativ via wg-mesh direkt:
curl -s http://<LLM_HOST_IP>:8081/health

# LLM_ENABLED im ConfigMap prüfen
kubectl get configmap -n workspace workspace-config -o yaml | grep LLM

Wenn LLM_ENABLED=false: Nur voyage-Pfad funktioniert. bge-m3-Collections können nicht indiziert werden.

2 Collection-Modell prüfen Welche Collection nutzt welches Modell? 
task workspace:psql ENV=<env> -- website

-- Modell-Zuordnung prüfen
SELECT collection_name, embedding_model
FROM knowledge.collections
ORDER BY collection_name;

-- Chunk-Status pro Collection
SELECT c.collection_name, c.embedding_model,
       COUNT(*) as chunks, COUNT(embedding) as indexed
FROM knowledge.chunks ch
JOIN knowledge.collections c ON c.id = ch.collection_id
GROUP BY c.collection_name, c.embedding_model;
3 Re-Indexing ausführen Logs beobachten 
task knowledge:reindex ENV=mentolder

# Logs der Indexing-Pods
kubectl logs -n workspace -l app=knowledge-indexer -f
