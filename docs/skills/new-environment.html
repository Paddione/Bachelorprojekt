Infrastruktur new-environment

Neue Umgebung aufsetzen

Vollständiges Aufsetzen einer neuen Cluster-Umgebung von Grund auf. Die Reihenfolge der Schritte ist zwingend — jede Abweichung führt zu stillen Credential-Überschreibungen oder nicht-funktionierenden Services.

Auslöser: Neuer Cluster oder neue Umgebung von Grund auf aufsetzen
⚠
KRITISCHE REIHENFOLGE — nicht überspringen. Jeden Schritt in exakter Reihenfolge ausführen. Ein übersprungener Schritt führt zu stillen Credential-Überschreibungen. SealedSecrets-Keypair-Rotation ist nach jedem Cluster-Reset zu erwarten — alte Sealed-Files entschlüsseln danach nicht mehr.
Pflicht-Reihenfolge
0 Env-Config scaffolden Grundkonfiguration 
# Neue Umgebung aus Schema scaffolden
task env:init ENV=<neue-umgebung>

# Pflichtfelder ausfüllen:
# - PROD_DOMAIN (z.B. "newenv.de")
# - BRAND_NAME (z.B. "My Company")
# - ENV_CONTEXT (kubectl-Kontext-Name)
# - ENV_OVERLAY (z.B. "prod-newenv")
# - CONTACT_EMAIL, LEGAL_* Felder

# Validieren
task env:validate ENV=<neue-umgebung>
task env:show ENV=<neue-umgebung>
1 Sealed Secrets Controller installieren Muss laufen bevor irgendein SealedSecret angewendet wird
⚠
Controller MUSS existieren bevor SealedSecrets angewendet werden. Reihenfolge: install → fetch-cert → seal → deploy. 
task sealed-secrets:install ENV=<env>

# Warten bis Pod Running
kubectl get pods -n kube-system -l app.kubernetes.io/name=sealed-secrets -w
2 Sealing-Certificate holen Cert des neuen Controllers fetchen und committen 
task env:fetch-cert ENV=<env>

# Cert landet in environments/certs/<env>.pem
# Committen!
git add environments/certs/<env>.pem
git commit -m "chore: add sealing cert for <env>"
⚠
Nach jedem Cluster-Reset muss dieses Cert neu geholt werden — Keypair-Rotation ist das Standardverhalten.
3 Secrets generieren Plaintext — gitignored 
task env:generate ENV=<env>

Alle MANAGED_EXTERNALLY-Placeholder manuell ersetzen:

  • Signaling/TURN-Secrets für Nextcloud Talk HPB
  • SMTP-Credentials
  • IPV64 API-Key
  • Anthropic API-Key (falls LLM aktiviert)
⚠
talk-hpb-setup.sh bricht ab wenn MANAGED_EXTERNALLY-Werte noch vorhanden sind.
4 Secrets versiegeln SealedSecret erstellen und committen 
task env:seal ENV=<env>

# environments/sealed-secrets/<env>.yaml wurde erstellt/aktualisiert
git add environments/sealed-secrets/<env>.yaml
git commit -m "chore: seal secrets for <env>"
git push
5 cert-manager installieren Vor workspace:deploy zwingend 
# cert-manager CRDs installieren
task cert:install ENV=<env>

# IPV64 API-Key als Secret speichern
# (in BEIDEN Namespaces: cert-manager UND workspace)
task cert:secret -- <ipv64-api-key> ENV=<env>

# Status prüfen
task cert:status ENV=<env>
6 Workspace deployen SealedSecrets + Kustomize-Overlay anwenden 
task workspace:deploy ENV=<env>

# Status beobachten
kubectl get pods -n workspace -w

# Logs bei Fehlern
task workspace:logs ENV=<env> -- <service>
7 Post-Deploy Setup Services konfigurieren

In dieser Reihenfolge ausführen:

task workspace:office:deploy ENV=<env>
task workspace:post-setup ENV=<env>
task workspace:talk-setup ENV=<env>
task workspace:recording-setup ENV=<env>
task workspace:whiteboard-setup ENV=<env>
task workspace:systembrett-setup ENV=<env>
task workspace:admin-users-setup ENV=<env>
task workspace:theme ENV=<env>
8 Verifizieren Smoke-Tests 
task workspace:verify ENV=<env>
task workspace:status ENV=<env>
task workspace:dsgvo-check ENV=<env>
9 Keycloak Realm synchronisieren Clients + Mapper konfigurieren 
task keycloak:sync ENV=<env>
bash scripts/keycloak-ensure-mappers.sh <env>

Realm-JSON Pfade:

  • dev: k3d/realm-workspace-dev.json
  • mentolder: prod-mentolder/realm-workspace-mentolder.json
  • korczewski: prod-korczewski/realm-workspace-korczewski.json
Bekannte Fallstricke
FallstrickSymptomLösung
knowledge-secrets Konflikt Controller refused to adopt Secret kubectl delete secret knowledge-secrets -n $NS
$patch:delete fehlt Prod-Secrets mit Dev-Werten überschrieben Niemals den $patch: delete Block in prod/kustomization.yaml entfernen
env:fetch-cert vergessen nach Reset SealedSecret entschlüsselt nicht env:fetch-cert → env:seal → deploy
Collabora fehlt Office in Nextcloud funktioniert nicht task workspace:office:deploy separat ausführen
