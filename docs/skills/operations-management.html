Incident Response & Repo Hygiene

Production incident triage (scope, diagnose, rollback or fix), DB ticket management (triage, close, route), git repo hygiene (prune worktrees & branches), PR reviews, and mishap tracking.

Auslöser: Service out or degraded, ticket queue triage, or daily git housekeeping.
ℹ️
Incident Triage: Answer service, cluster, start time, and blast radius within 2 minutes of discovering a failure.
1 Incident Scope Open a Bug Ticket

Determine scope and register a bug ticket inside the tickets schema:

INSERT INTO tickets.tickets (type, status, severity) VALUES ('bug', 'in_progress', 'critical');
2 Fix or Rollback Restore Stability

If introduced by the latest deploy (no migrations), roll back container image digests:

kubectl set image deployment/website website=ghcr.io/paddione/workspace-website:<PREV_SHA>
3 Ticket Triage Work the Queue

Fetch open tickets, classify them (AI-actionable, human needed, obsolete, or resolved), and resolve/update notes accordingly.

4 Git & PR Hygiene Cleanup Resources

Prune stale branches and remove merged git worktrees to keep the repository clean:

git worktree remove /tmp/wt-<slug> --force
git fetch --prune
