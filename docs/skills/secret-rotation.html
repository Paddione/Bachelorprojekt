Secret-Rotation

Sicher rotieren von Credentials, SealedSecrets-Keypairs und Service-Tokens. Die Reihenfolge ist absolut zwingend — kein Deploy vor dem Seal. Beide Cluster immer separat behandeln.

Auslöser: Credential-Verdacht, Cluster-Reset, regelmäßige Rotation, DB-Passwort-Drift
⚠
KRITISCHE REIHENFOLGE — kein Abweichen. NIEMALS workspace:deploy vor env:seal. Dev-Placeholder überschreiben sonst Prod-Secrets. Pflichtabfolge: sealed-secrets:install → env:fetch-cert → env:generate → env:seal → workspace:deploy
  • Beide ClusterImmer separat — jeder hat eigenen Controller, Cert, shared-db
  • Secrets-Quelleenvironments/.secrets/<env>.yaml (gitignored)
  • Sealed Outputenvironments/sealed-secrets/<env>.yaml (committed)
5 Rotationstypen
A DB-Passwort-Drift korrigieren Wenn PG-Rolle vom SealedSecret abgewichen ist

Symptom: Service-Pods zeigen "password authentication failed"

# Auf beiden Clustern!
task workspace:sync-db-passwords ENV=mentolder
task workspace:sync-db-passwords ENV=korczewski

# Alternativ manuell:
task workspace:psql ENV=mentolder -- postgres
-- ALTER ROLE website WITH PASSWORD '<neues-passwort>';
B Neue Secrets generieren & versiegeln Vollständige Rotation aller Secrets 
# Schritt 1: Neue Secrets generieren
task env:generate ENV=<env>
# MANAGED_EXTERNALLY-Placeholder manuell ersetzen

# Schritt 2: Versiegeln
task env:seal ENV=<env>
git add environments/sealed-secrets/<env>.yaml
git commit -m "chore: rotate secrets for <env>"

# Schritt 3: Auf Cluster anwenden (OHNE workspace:deploy)
task secrets:sync ENV=<env>

# Schritt 4: Betroffene Workloads neu starten
kubectl rollout restart deployment/<service> -n workspace
⚠
secrets:sync wendet SealedSecrets an ohne Workloads zu rollen. Workloads explizit neu starten wenn Secrets-Änderungen wirksam werden sollen.
C SealedSecrets Keypair-Refresh nach Cluster-Reset Zwingend nach jedem Cluster-Reset
⚠
Nach Cluster-Reset: Neues Keypair → alte SealedSecrets entschlüsseln nicht mehr. Diese Sequenz MUSS eingehalten werden. 
# 1. Controller installieren
task sealed-secrets:install ENV=<env>

# 2. Auf Running warten
kubectl get pods -n kube-system -l app=sealed-secrets -w

# 3. Neues Cert holen
task env:fetch-cert ENV=<env>
git add environments/certs/<env>.pem
git commit -m "chore: update sealing cert after cluster reset"

# 4. Mit neuem Cert versiegeln
task env:seal ENV=<env>
git add environments/sealed-secrets/<env>.yaml
git commit -m "chore: reseal secrets for <env>"

# 5. Deployen
task workspace:deploy ENV=<env>
