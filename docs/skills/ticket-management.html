Ticket Management

Verwaltung und Abarbeitung von Tickets in der Plattform-Datenbank sowie Repo-Hygiene: Stale Branches, Worktrees und PR-Triage.

Auslöser: Offene Bug-Reports, Feature-Tasks oder notwendige Repo-Bereinigung
0 Repo Hygiene Immer zuerst ausführen

Bereinige verwaiste Worktrees und gelöschte Remote-Branches:

git worktree prune
git fetch --prune
git branch --merged main | xargs git branch -d
1 Ticket Triage Abruf & Kategorisierung

Hole alle offenen Tickets aus der shared-db:

SELECT external_id, title, status, priority
FROM tickets.tickets
WHERE status NOT IN ('done', 'archived');
2 Execution AI-Fix vs. Human-Needed

Klassifiziere jedes Ticket:

  • Already resolved: Direkt schließen.
  • AI-ready: Automatisch fixen & PR erstellen.
  • Needs Human: Spezifische Frage an den Nutzer formulieren.
3 Abschluss Ticketing Etiquette

Schließe Tickets immer mit Datum, Resolution und kurzer Note:

UPDATE tickets.tickets SET
  status = 'done', resolution = 'fixed',
  notes = '[ticket-management] Fixed in PR #123';
