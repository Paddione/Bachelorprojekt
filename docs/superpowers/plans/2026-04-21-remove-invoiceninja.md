> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Tech Stack:** Kubernetes, Keycloak, kubeseal, kubectl (context: mentolder), bash

## Task 14: Delete invoiceninja OIDC client from live Keycloak

The realm JSON change only affects new cluster imports. The live Keycloak database still has the `invoiceninja` client — delete it via the admin API.

- [ ] **Step 1: Get admin password from workspace-secrets**

```bash
KC_PASS=$(kubectl --context mentolder get secret workspace-secrets -n workspace \
  -o jsonpath='{.data.KEYCLOAK_ADMIN_PASSWORD}' | base64 -d)
KC_URL="https://auth.mentolder.de"
```

- [ ] **Step 2: Get admin token**

```bash
TOKEN=$(curl -s -X POST "${KC_URL}/realms/master/protocol/openid-connect/token" \
  -d "client_id=admin-cli&grant_type=password&username=admin&password=${KC_PASS}" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo "Token acquired: ${TOKEN:0:20}..."
```

- [ ] **Step 3: Find the invoiceninja client UUID**

```bash
CLIENT_UUID=$(curl -s -H "Authorization: Bearer $TOKEN" \
  "${KC_URL}/admin/realms/workspace/clients?clientId=invoiceninja" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d[0]['id'] if d else 'NOT_FOUND')")
echo "InvoiceNinja client UUID: $CLIENT_UUID"
```

If output is `NOT_FOUND`, the client was never created (skip Step 4).

- [ ] **Step 4: Delete the client**

```bash
curl -s -X DELETE -H "Authorization: Bearer $TOKEN" \
  "${KC_URL}/admin/realms/workspace/clients/${CLIENT_UUID}"
echo "Deleted invoiceninja OIDC client"
```

- [ ] **Step 5: Verify it's gone**

```bash
curl -s -H "Authorization: Bearer $TOKEN" \
  "${KC_URL}/admin/realms/workspace/clients?clientId=invoiceninja" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print('GONE' if not d else 'STILL EXISTS:', d)"
```

Expected: `GONE`
