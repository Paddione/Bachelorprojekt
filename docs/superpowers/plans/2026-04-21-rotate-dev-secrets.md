# Rotate Dev Secrets in Live mentolder Cluster

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove all dev placeholder secrets from the live mentolder cluster, remove unused Grafana and Mattermost references, and ensure the environment is secure.

**Architecture:** Three secrets surfaces: (1) `workspace-secrets` managed via SealedSecret in `environments/sealed-secrets/mentolder.yaml`, (2) `website-secrets` managed as a separate SealedSecret in the `website` namespace (currently not tracked in git — will be added), (3) `claude-code-secrets` applied directly via kubectl. Traefik basic-auth is a fourth surface managed inline via a prod patch.

**Tech Stack:** kubeseal, kubectl (context: mentolder), openssl, htpasswd (apache2-utils)

## Task 6: Create website-secrets SealedSecret

The live `website-secrets` (namespace: website) has `WEBSITE_DB_PASSWORD=devwebsitedb` and `MATTERMOST_BOT_TOKEN` set. It is a SealedSecret with no git source — we add one now.

- [ ] **Step 1: Decode current values we want to preserve**

```bash
kubectl --context mentolder get secret website-secrets -n website -o json | \
  python3 -c "import sys,json,base64; d=json.load(sys.stdin)['data']; [print(k+': '+base64.b64decode(v).decode()) for k,v in sorted(d.items())]"
```

- [ ] **Step 2: Build new website-secrets manifest**

Create `/tmp/website-secrets.yaml` with the following (substituting real values from Step 1 except WEBSITE_DB_PASSWORD and omitting MATTERMOST_BOT_TOKEN):

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: website-secrets
  namespace: website
type: Opaque
stringData:
  CRON_SECRET: "<value from workspace-secrets>"
  NEXTCLOUD_ADMIN_PASS: "<NEXTCLOUD_ADMIN_PASSWORD from workspace-secrets>"
  NEXTCLOUD_CALDAV_PASSWORD: "<NEXTCLOUD_ADMIN_PASSWORD from workspace-secrets>"
  SELLER_ADDRESS: "Ludwig-Erhard-Str. 18"
  SELLER_CITY: "Hamburg"
  SELLER_COUNTRY: "DE"
  SELLER_NAME: "Gerald Korczewski"
  SELLER_POSTAL_CODE: "20459"
  SELLER_VAT_ID: ""
  SMTP_PASS: "<SMTP_PASSWORD from workspace-secrets>"
  STRIPE_PUBLISHABLE_KEY: "<current value from live secret>"
  STRIPE_SECRET_KEY: "<current value from live secret>"
  STRIPE_WEBHOOK_SECRET: "<current value from live secret>"
  WEBSITE_DB_PASSWORD: "<new WEBSITE_DB_PASS generated in Task 1>"
  WEBSITE_OIDC_SECRET: "<current value from live secret>"
```

- [ ] **Step 3: Seal with kubeseal**

```bash
kubeseal --cert environments/certs/mentolder.pem \
         --format yaml \
         < /tmp/website-secrets.yaml \
         > environments/sealed-secrets/mentolder-website.yaml
rm /tmp/website-secrets.yaml
```

## Task 7: Add prod patch for traefik-basic-auth

- [ ] **Step 1: Generate htpasswd hash**

```bash
htpasswd -nb admin "$TRAEFIK_PASS"
# Outputs something like: admin:$apr1$xxxxx$xxxxxxxxxxxxxxxxxxxxxxxxxx
```

- [ ] **Step 2: Base64-encode the hash**

```bash
echo -n 'admin:$apr1$xxxxx$xxxxxxxxxx' | base64 -w0
```

- [ ] **Step 3: Create prod/patch-traefik-basic-auth.yaml**

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: traefik-basic-auth
  namespace: workspace
type: Opaque
data:
  users: <base64-of-htpasswd-hash>
```

- [ ] **Step 4: Add patch to prod/kustomization.yaml**

Add to the `patches:` list:
```yaml
  - path: patch-traefik-basic-auth.yaml
```

## Task 12: Update claude-code-secrets on live cluster

- [ ] **Step 1: Get live key values needed**

```bash
SHARED_DB=$(kubectl --context mentolder get secret workspace-secrets -n workspace -o jsonpath='{.data.SHARED_DB_PASSWORD}' | base64 -d)
STRIPE_LIVE=$(kubectl --context mentolder get secret workspace-secrets -n workspace -o jsonpath='{.data.STRIPE_SECRET_KEY}' | base64 -d)
IN_TOKEN=$(kubectl --context mentolder get secret workspace-secrets -n workspace -o jsonpath='{.data.INVOICENINJA_API_TOKEN}' | base64 -d)
NC_PASS=$(kubectl --context mentolder get secret workspace-secrets -n workspace -o jsonpath='{.data.NEXTCLOUD_ADMIN_PASSWORD}' | base64 -d)
```

- [ ] **Step 2: Apply updated claude-code-secrets**

```bash
kubectl --context mentolder apply -f - <<EOF
apiVersion: v1
kind: Secret
metadata:
  name: claude-code-secrets
  namespace: workspace
type: Opaque
stringData:
  GITHUB_PAT: "<production-pat-from-vaultwarden>"
  INVOICENINJA_REQUEST_HEADERS: "X-Api-Token: ${IN_TOKEN}"
  NEXTCLOUD_PASSWORD: "${NC_PASS}"
  NEXTCLOUD_USERNAME: "admin"
  SHARED_DB_PASSWORD: "${SHARED_DB}"
  STRIPE_SECRET_KEY: "${STRIPE_LIVE}"
EOF
```

Note: MATTERMOST_TOKEN is intentionally omitted (not used).
