# Filen Remote Backup Target Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Filen WebDAV as a second backup target alongside the existing local PVC, with per-cluster upload paths configurable at runtime via the admin panel.

**Architecture:** A second `filen-upload` container (rclone/rclone image) is added to the existing CronJob pod. The backup container writes encrypted `.enc` files to the local PVC as before, then writes two signal files to a shared `staging` emptyDir: `.done` (the timestamp stamp) and `.filen_path` (the upload path read from the `site_settings` DB table). The rclone container waits for `.done`, then uploads from the PVC to Filen WebDAV. Credentials live in `workspace-secrets` (sealed). The upload path default lives in a new per-cluster `backup-config` ConfigMap; the admin panel writes overrides to `site_settings` so the path is runtime-changeable without redeployment.

**Tech Stack:** Kubernetes CronJob, rclone WebDAV, Filen WebDAV (`https://webdav.filen.io`), SealedSecrets, Kustomize strategic-merge patches, Astro + Svelte admin panel, PostgreSQL `site_settings` table.

## Task 4: Update backup CronJob with Filen upload

- [ ] **Step 3: Append Filen path resolution + signal files to the backup script**

  In the `args:` shell script, replace the final two lines:

  ```bash
                  echo "Backup complete: ${BACKUP_DIR}"
                  ls -lh "${BACKUP_DIR}"/
  ```

  with:

  ```bash
                  # Resolve runtime upload path: DB value wins, ConfigMap default is fallback
                  FILEN_PATH=$(PGPASSWORD="${WEBSITE_DB_PASSWORD}" \
                    psql -h shared-db -U website -d website -t -q \
                    -c "SELECT value FROM site_settings WHERE brand='${BRAND}' AND key='filen_upload_path'" \
                    2>/dev/null | tr -d ' \n')
                  FILEN_PATH="${FILEN_PATH:-${FILEN_DEFAULT_UPLOAD_PATH}}"

                  # Signal filen-upload container
                  printf '%s' "${FILEN_PATH}" > /staging/.filen_path
                  printf '%s' "${STAMP}"      > /staging/.done

                  echo "Backup complete: ${BACKUP_DIR}"
                  ls -lh "${BACKUP_DIR}/"
  ```

- [ ] **Step 4: Add filen-upload container after the backup container's closing brace**

  After the `backup` container's `resources:` block and before `volumes:`, add a new entry in the `containers:` list:

  ```yaml
            - name: filen-upload
              image: rclone/rclone:1.68
              imagePullPolicy: IfNotPresent
              securityContext:
                allowPrivilegeEscalation: false
                capabilities:
                  drop: ["ALL"]
              command: ["/bin/sh", "-c"]
              args:
                - |
                  if [ -z "${FILEN_EMAIL}" ] || [ -z "${FILEN_PASSWORD}" ]; then
                    echo "Filen not configured — skipping remote backup"
                    until [ -f /staging/.done ]; do sleep 2; done
                    exit 0
                  fi

                  echo "Waiting for local backup to complete..."
                  until [ -f /staging/.done ]; do sleep 2; done

                  STAMP=$(cat /staging/.done)
                  UPLOAD_PATH=$(cat /staging/.filen_path)

                  echo "Uploading ${STAMP} to Filen: ${UPLOAD_PATH}/${STAMP}/"

                  OBSCURED=$(rclone obscure "${FILEN_PASSWORD}")
                  export RCLONE_WEBDAV_URL=https://webdav.filen.io
                  export RCLONE_WEBDAV_USER="${FILEN_EMAIL}"
                  export RCLONE_WEBDAV_PASS="${OBSCURED}"

                  rclone copy "/backups/${STAMP}/" ":webdav:${UPLOAD_PATH}/${STAMP}/" \
                    --no-traverse \
                    --stats-one-line \
                    || echo "WARNING: Filen upload failed — local backup intact"

                  echo "Filen upload done"
              env:
                - name: FILEN_EMAIL
                  valueFrom:
                    secretKeyRef:
                      name: workspace-secrets
                      key: FILEN_EMAIL
                      optional: true
                - name: FILEN_PASSWORD
                  valueFrom:
                    secretKeyRef:
                      name: workspace-secrets
                      key: FILEN_PASSWORD
                      optional: true
              volumeMounts:
                - name: backup-storage
                  mountPath: /backups
                  readOnly: true
                - name: staging
                  mountPath: /staging
                  readOnly: true
              resources:
                requests:
                  memory: 64Mi
                  cpu: "50m"
                limits:
                  memory: 128Mi
                  cpu: "200m"
  ```

- [ ] **Step 3: Create website/src/pages/api/admin/einstellungen/backup.ts**

  ```typescript
  import type { APIRoute } from 'astro';
  import { getSession, isAdmin } from '../../../../lib/auth';
  import { setSiteSetting } from '../../../../lib/website-db';

  export const POST: APIRoute = async ({ request, redirect }) => {
    const session = await getSession(request.headers.get('cookie'));
    if (!session || !isAdmin(session)) return new Response('Forbidden', { status: 403 });

    const form = await request.formData();
    const brand = process.env.BRAND || 'mentolder';

    const uploadPath = (form.get('filen_upload_path') as string)?.trim();

    if (!uploadPath || !uploadPath.startsWith('/')) {
      return new Response(JSON.stringify({ error: 'Pfad muss mit / beginnen' }), { status: 400 });
    }

    await setSiteSetting(brand, 'filen_upload_path', uploadPath);

    return redirect('/admin/einstellungen/backup?saved=1', 303);
  };
  ```

## Notes

- **Dev has no Filen backup**: `FILEN_EMAIL`/`FILEN_PASSWORD` are absent from `k3d/secrets.yaml`. The `optional: true` flag on `secretKeyRef` means the pod starts fine with empty env vars, and both containers skip the upload gracefully.
- **Upload failure is non-fatal**: the `filen-upload` container exits 0 even on rclone failure (warning printed). The local PVC backup is always the primary.
- **ReadWriteOnce on backup-pvc**: both containers can share the PVC because they run on the same pod (same node). The RWO restriction only prevents multi-node access.
- **Retention on Filen**: rclone `copy` only uploads — it does NOT delete old remote backups. Filen acts as an append-only archive. Manual cleanup or a retention job can be added later if needed.
- **Password reuse**: `FILEN_PASSWORD` reuses the SMTP password value (`Plotterpapier11!$`). This is fine for now but worth noting for future credential rotation.
