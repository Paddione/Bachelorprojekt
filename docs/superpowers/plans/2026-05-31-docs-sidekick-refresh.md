# Docs site refresh + Hilfesidekick skill-init prompts — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Author skill-initialization prompts once in the agent-guide registry so they flow to both the docs site and the website Hilfesidekick, and give `docs.{mentolder,korczewski}.de` an on-brand dark reskin plus a few stale-content fixes.

**Architecture:** `docs/agent-guide/registry/*.yaml` is the single source of truth; three emitters (`emit-docs.mjs`, `emit-maps.mjs`, `emit-webapp.mjs`) project it to docs Markdown, agent-readable maps, and the in-app JSON the sidekick consumes. A new optional `init_prompt_de` field rides that pipeline. The docs HTML is generated by `scripts/docs-gen/` (`theme.mjs` = CSS, `templates.mjs` = page chrome); the reskin recolours via CSS variables only (mermaid snapshots are SHA-cached — never re-render them).

**Tech Stack:** Node.js ESM (`node:test`), YAML registry, Svelte 5 (runes) sidekick components, Playwright E2E, static HTML docs generator.

**Packaging — one plan, two PRs (independently shippable, different deploy paths):**
- **PR 1 — Workstream A (Tasks 1–10):** registry + emitters + sidekick UI → ships via the **website** build + rollout.
- **PR 2 — Workstream B (Tasks 11–18):** content fixes + dark reskin → ships via **docs image rebuild + `task docs:deploy`** (rolls out to both `workspace` + `workspace-korczewski` on the fleet context; `docs:sync` is dead — read-only rootfs).

**Spec corrections applied in this plan (verified against the tree):**
1. The docs glossary the spec calls `docs/glossary.md` does **not** exist; the rendered glossary is `docs/legacy-html/glossary.html` (Task 12 targets it).
2. There is **no** Google-Fonts `<link>` or CSS `@import` to "replace" — `theme.mjs` references fonts only via `--font-sans`/`--font-serif` with system fallbacks. Geist + Instrument Serif are not system fonts, so Task 15 **adds** a font `<link>` in `documentHead()`.
3. `scripts/docs-gen/theme.test.mjs` currently asserts the CSS contains `Inter` + `Merriweather`; Task 14 updates that test to the new fonts (TDD red→green) — do not skip it or CI breaks.

**Working directory:** isolated worktree `/tmp/wt-docs-sidekick-refresh` on branch `feature/docs-sidekick-refresh`. Pull --rebase before starting: `git -C /tmp/wt-docs-sidekick-refresh pull --rebase origin feature/docs-sidekick-refresh`.
