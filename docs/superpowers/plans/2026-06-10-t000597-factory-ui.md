## Ziel

Token-Library und vier wiederverwendbare Svelte-Basiskomponenten erstellen, die das Industrial/Loft Design System der Factory-UI-Suite definieren. Dieses Fundament muss vor T000598–T000601 gemergt sein.

---

## Architektur

### Neue Dateien

```
website/src/styles/factory-tokens.css          # CSS Custom Properties (alle Design-Tokens)
website/src/components/factory/
  PilotLight.svelte                            # Status-Indikator (green/amber/red Glow)
  WorkpieceCard.svelte                         # Ticket-Karte mit Priority-LED-Strip
  PhaseBadge.svelte                            # Phase-Pill-Badge
  ViewSwitcher.svelte                          # Viewport-Auto-Detect + localStorage Override
  index.ts                                     # Re-Export aller Komponenten
website/src/pages/factory/design-system.astro  # Demo/Dev-Seite fuer alle Komponenten
```


## Tasks

- [x] **T1 — Token-Stylesheet anlegen:** `website/src/styles/factory-tokens.css` mit allen 11 Farb-Tokens, Priority-Map, Typografie-Tokens, Abstands-Tokens und Grid-Overlay-Klasse erstellen.

- [x] **T2 — Global-Import:** `factory-tokens.css` in `website/src/styles/global.css` (oder dem verwendeten Global-Entry) via `@import` einbinden, sodass die Tokens auf allen Seiten verfuegbar sind.

- [x] **T3 — PilotLight.svelte:** Komponente mit Props `state`, `label`, `size`, `animated` implementieren. Glow via `box-shadow` mit passender Farbe. Unit-Test oder Sicht-Pruefung auf Demo-Seite.

- [x] **T4 — WorkpieceCard.svelte:** Komponente mit `ticket`-Prop (TypeScript-Interface `FactoryTicket` definieren) und `compact`-Flag implementieren. 4 px linker Border via Priority-Farb-Map. Hover-State.

- [x] **T5 — PhaseBadge.svelte:** Pill-Badge mit `phase`-Prop. Bekannte Phasen (`scout`, `plan`, `implement`, `review`, `done`, `blocked`) bekommen eigene Farbe; unbekannte Phasen erhalten `--factory-gray`. Monospace, 11 px.

- [x] **T6 — ViewSwitcher.svelte:** Viewport-Auto-Detection via `window.matchMedia('(max-width: 768px)')`. localStorage-Key (`factory-view`) lesen/schreiben beim Toggle. Custom-Event `viewchange` emittieren. SSR-safe (kein direkter `window`-Zugriff beim Render).

- [x] **T7 — index.ts Re-Export:** `website/src/components/factory/index.ts` anlegen, der alle vier Komponenten re-exportiert, sodass Consumer `import { PilotLight, WorkpieceCard } from '$components/factory'` schreiben koennen.

- [x] **T8 — Demo-Seite:** `website/src/pages/factory/design-system.astro` erstellen mit je einem Beispiel aller Komponenten in allen relevanten States/Props-Kombinationen. Seite ist NICHT SSO-gesperrt (Dev-only, kein Link in Nav).

- [x] **T9 — TypeScript-Check:** `cd website && pnpm tsc --noEmit` ausfuehren; alle Typ-Fehler beheben.

- [x] **T10 — Mobile-Pruefung:** Browser DevTools auf 375 px (iPhone SE) — alle Komponenten sichtbar, kein horizontales Overflow, WorkpieceCard zeigt alle Pflichtfelder.

- [x] **T11 — CI-Smoke:** `task test:all` lokal gruener als vor diesem Branch. Mindestens `task workspace:validate` ohne Fehler.
