# Plan: Factory UI — Analytics Dashboard (Charts + Heatmap)

**Ticket:** T000600  
**Datum:** 2026-06-10  
**Branch:** feature/t000600  
**Spec:** docs/superpowers/specs/2026-06-10-t000600-factory-ui-design.md

---

## Ziel

Analytics-Tab in `/dev-status` mit KPI-Kacheln, Throughput-Chart (7 Tage), Phase-Heatmap und Shipped-Bar-Chart ausbauen. Chart.js für Canvas-Charts, SVG direkt für die Heatmap. Volle Mobile-Parität, Industrial/Loft-Ästhetik.

---

## Architektur

### Neue Dateien

```
website/src/components/factory/
  FactoryKpiCard.svelte          # Eine KPI-Kachel (icon, value, label, trend)
  FactoryKpiGrid.svelte          # 5-Kacheln-Grid, lädt KPI-Daten aus factory-metrics
  FactoryThroughputChart.svelte  # Chart.js LineChart (filled area, 7 Tage)
  FactoryPhaseHeatmap.svelte     # SVG 7x6 Heatmap (Tage x Phasen)
  FactoryShippedBar.svelte       # Chart.js HorizontalBar (Phasen farbkodiert)
  factory-chart-colors.ts        # Farbkonstanten + Interpolations-Helper
```

### Geänderte Dateien

```
website/src/pages/admin/dev-status.astro
  # Analytics-Tab: importiert + rendert alle 4 neuen Komponenten

website/src/styles/factory.css (oder tailwind-Klassen inline)
  # Ergänzung: KPI-Grid, Heatmap-Container, mobile Breakpoints

website/package.json
  # chart.js als Dependency hinzufügen (falls nicht vorhanden)
```

### Keine Änderungen an

- Backend / API-Routes (bestehende `/api/factory-metrics` und `/api/factory-floor` werden as-is konsumiert)
- Datenbankschema
- Kubernetes-Manifeste
- `environments/schema.yaml`

## Tech-Stack

| Dependency | Version | Zweck |
|------------|---------|-------|
| `chart.js` | ^4.x | Throughput-LineChart + Shipped-BarChart |
| Svelte (bestehend) | ^5.x | Komponenten-Framework |
| Astro (bestehend) | ^5.x | Seiten-Framework, SSR |
| SVG inline | — | Phase-Heatmap (kein zusätzliches Paket) |
| Heroicons SVG | inline | KPI-Icons (kein npm-Paket nötig) |

Chart.js wird als Client-side-only Import eingebunden (`<script>` im Svelte `onMount`), um SSR-Konflikte mit `document`/`canvas` zu vermeiden.

---

## Tasks

- [x] **T1 — Abhängigkeit prüfen + Chart.js installieren**  
  `pnpm add chart.js` im `website/`-Verzeichnis; prüfen ob chart.js bereits in package.json; sicherstellen dass kein Versionskonflikt mit T000597-Design-System entsteht.

- [x] **T2 — Farbkonstanten-Modul anlegen**  
  `factory-chart-colors.ts` mit allen Design-Token-Konstanten (Background, Surface, Border, Amber, Terminal-Green, Heatmap-Min/Max, Phasenfarben-Array) und einem linearen Interpolations-Helper für die Heatmap-Zellenfarbe.

- [x] **T3 — FactoryKpiCard.svelte implementieren**  
  Einzelne Kachel: `#1e2736` Hintergrund, `1px #2d3748` Border, icon-Slot (SVG inline), Wert (`#ffffff`, 32px bold), Label (Monospace, `uppercase`, `#6b7280`). Optionaler `trend`-Prop (positiv/negativ Pfeil).

- [x] **T4 — FactoryKpiGrid.svelte implementieren**  
  Ruft `/api/factory-metrics` auf, mapped MetricRow[] auf die 5 KPI-Definitionen, rendert 5× `FactoryKpiCard`. Desktop: 5-Spalten-Grid; Mobile (< 640px): 2-Spalten. Fehlerzustand: Skeleton-Kacheln bei Ladefehler.

- [x] **T5 — FactoryThroughputChart.svelte implementieren**  
  Chart.js `LineChart` (`type: 'line'`, `fill: true`) in `onMount`, destroy in `onDestroy`. Daten: letzte 7 Tage aus `/api/factory-metrics` (throughput-Feld). Farbe `#22c55e`, Grid `rgba(255,255,255,0.05)`, Achsen Monospace. `canvas`-Element mit `width: 100%`, `aspect-ratio: 16/6` auf Desktop, `16/9` auf Mobile.

- [x] **T6 — FactoryPhaseHeatmap.svelte implementieren**  
  SVG-Grid 7 Spalten (Mo–So) × 6 Zeilen (Phasennamen). Daten aus `/api/factory-floor` (Stunden pro Phase/Tag). Farbinterpolation via Helper aus T2: 0h → `#1a2634`, Maximum → `#f59e0b`. Tooltip via `<title>` auf jedem `<rect>`. Horizontaler Scroll-Container auf Mobile.

- [x] **T7 — FactoryShippedBar.svelte implementieren**  
  Chart.js `BarChart` (`indexAxis: 'y'`) in `onMount`/`onDestroy`. Phasen als Y-Achse, shipped-Count als X-Achse. Phasenfarben aus Farb-Array (T2). Legende deaktiviert (Phasennamen direkt als Labels). Mobile: `height` dynamisch nach Anzahl Phasen.

- [x] **T8 — dev-status.astro Analytics-Tab verdrahten**  
  Alle 4 Komponenten (`FactoryKpiGrid`, `FactoryThroughputChart`, `FactoryPhaseHeatmap`, `FactoryShippedBar`) in den Analytics-Tab importieren und rendern. Reihenfolge: KPI-Grid → Throughput → Heatmap → Shipped. `client:load` Direktive für Chart.js-Komponenten.

- [x] **T9 — Mobile-CSS / Responsive-Anpassungen**  
  Sicherstellen dass alle Charts auf 390px Portrait korrekt skalieren: `width: 100%`, kein horizontaler Überlauf außer Heatmap (expliziter scroll-container), KPI-Grid 2-Spalten, vertikales Stacking der Chart-Sektionen.

- [x] **T10 — Fehlerzustände + Loading-States**  
  Skeleton-Platzhalter (pulsing `#1e2736`-Blöcke) während Daten laden. Bei API-Fehler: Fehlermeldung im Chart-Container statt leerem Canvas. `console.error` nur bei echten Fehlern (nicht bei 404 auf leere Metriken).

- [x] **T11 — Manuelle Verifikation + CI-Check**  
  `pnpm build` in `website/` muss fehlerfrei durchlaufen. Kein TypeScript-Fehler (`pnpm typecheck`). Chrome DevTools: keine JS-Errors beim Öffnen des Analytics-Tabs auf Desktop und Mobile-Emulation (390px).
