# Plan: T000667 — Knowledge-Graph v2

**Ziel:** `scripts/build-graph.mjs` um vier neue Kantentypen erweitern (`ingress` via Traefik IngressRoute, `selector` via Service.spec.selector, `secret` via Shared-Secret-Pattern, `kind`-Feld auf allen Kanten), einen neuen `docs/generated/blast-radius.md`-Report generieren und beides in die Freshness-Pipeline integrieren.

**Spec:** `docs/superpowers/specs/2026-06-12-t000667-design.md`

**Architektur-Entscheidungen (aus Spec):**
- Secrets: keine eigenen Knoten; Kante nur bei ≥ 2 Consumern desselben Secrets.
- Beide Ingress-Varianten: vanilla `Ingress` (bereits partiell) + Traefik `IngressRoute` CRD.
- Ingress → Service → Workload via `selector`-Auflösung.
- `kind`-Feld additiv (abwärtskompatibel, bestehende `{from,to,via}` unberührt).
- Blast-Radius: transitive Downstream-Abhängige, BFS auf Reverse-Adjacency.
- PR-#1501-Muster: Regenerate + Commit im selben Schritt.

## Phase B — `build-graph.mjs` erweitern

- [x] **B1** — `kind`-Feld auf bestehende Kanten-Builder aufbauen.
  - In `extractEnvEdges`: `kind: 'env'` hinzufügen.
  - In `extractCommandEdges`: `kind: 'command'` hinzufügen (bzw. `'initContainer'` für initContainers).
  - In `extractAnnotationEdges`: `kind: 'annotation'` hinzufügen.
  - In ConfigMap-Pfad: `kind: 'configmap'` hinzufügen.
  - In bestehendem Ingress-Pfad (vanilla): `kind: 'ingress'` hinzufügen.
  - Edge-Deduplizierungs-Key erweitern: `${e.from}→${e.to}→${e.via}` bleibt (via ist bereits unique genug).

- [x] **B2** — Traefik `IngressRoute` CRD parsen.
  - In Pass 2, nach `kind === 'Ingress'`-Block: neuer Block für `kind === 'IngressRoute'`.
  - Pfad: `doc.spec.routes[]` → je Route `route.services[]` → `{ name, port }`.
  - Kante: `{ from: 'traefik', to: svc.name, via: 'ingress', kind: 'ingress' }`.
  - Namespace: aus `doc.metadata.namespace` ableiten.

- [x] **B3** — Service-Selector-Auflösung (neue Kette: Ingress → Service → Workload).
  - **Pass 1-Extension:** Alle `Service`-Objekte sammeln:
    ```
    Map<serviceName, { selector: Map<labelKey, labelValue> }>
    ```
  - **Pass 2-Extension:** Für jeden Workload-Node (Deployment/StatefulSet/CronJob):
    - Pod-Labels aus `spec.template.metadata.labels` lesen.
    - Für jeden bekannten Service prüfen: enthält `selector` ⊆ `pod-labels`?
    - Falls ja: Kante `{ from: serviceName, to: workloadName, via: 'selector', kind: 'selector' }`.
  - Traefik selbst bekommt KEINE selector-Kante (würde alle Services connecten).

- [x] **B4** — Shared-Secret-Kanten.
  - **Pass 1-Extension:** Für jede Workload: alle Secret-Referenzen sammeln.
    - `podSpec.volumes[].secret.secretName`
    - `container.env[].valueFrom.secretKeyRef.name`
    - `container.envFrom[].secretRef.name`
    - `imagePullSecrets` ignorieren.
    - `Map<secretName, Set<workloadId>>` aufbauen.
  - **Pass 3 (neu):** Für jedes Secret mit `Set.size >= 2`: alle Workload-Paare als Kanten ausgeben.
    - `{ from: wA, to: wB, via: 'secret:<secretName>', kind: 'secret' }`.
    - Symmetrisch: auch `{ from: wB, to: wA, ... }`.
    - Deduplizierung über bestehenden `edgeSet`.

- [x] **B5** — Sicherstellen: `traefik`-Knoten wird angelegt wenn IngressRoute-Kanten vorhanden.
  - Bestehende Logik prüft bereits `if (ingressEdges.length > 0 && !nodes.has('traefik'))`.
  - Analog für neu gesammelte IngressRoute-Kanten.

- [x] **B6** — Smoke-Test: `node scripts/build-graph.mjs`
  - Isolierte Knoten zählen: `node -e "const g=require('./docs/generated/graph.json'); const c=new Set([...g.edges.map(e=>e.from),...g.edges.map(e=>e.to)]); console.log(g.nodes.filter(n=>!c.has(n.id)).map(n=>n.id))"`
  - Ziel: ≤ 5 isolierte Knoten (von 14).
  - Alle Kanten haben `kind`-Feld: `node -e "const g=require('./docs/generated/graph.json'); console.log(g.edges.filter(e=>!e.kind).length, 'missing kind')"`

## Phase C — `build-blast-radius.mjs` (neues Script)

- [ ] **C1** — Script-Gerüst erstellen (`scripts/build-blast-radius.mjs`).
  - Liest `docs/generated/graph.json`.
  - Schreibt `docs/generated/blast-radius.md`.

- [ ] **C2** — Reverse-Adjacency-Liste aufbauen.
  - Für jeden Knoten: wer referenziert ihn als `to`? → `Map<nodeId, Set<nodeId>>` (directUpstream).
  - Für jeden Knoten: wen referenziert er als `to`? → `Map<nodeId, Set<nodeId>>` (directDownstream).

- [ ] **C3** — Transitive Closure (BFS downstream).
  - Funktion `getTransitiveDependents(nodeId)`: BFS über Reverse-Adjacency.
  - Cycle-Detection: besuchte Nodes tracken, keine Endlos-Schleife.
  - Gibt `Set<nodeId>` aller transitiven Abhängigen zurück.

- [ ] **C4** — Ranking berechnen.
  - Für alle Knoten (nur Workload-Typen: Deployment, StatefulSet, CronJob + Service-Pseudo-Knoten mit Edges):
    - `transitiveCount = getTransitiveDependents(id).size`
    - `directCount = directUpstream.get(id).size` (In-Degree)
  - Sortierung: absteigend nach `transitiveCount`, dann `directCount`.

- [ ] **C5** — Markdown-Report schreiben.
  ```markdown
  # Blast-Radius-Report
  > Generated: <ISO>  
  > Nodes: N | Edges: E | Isolated: I

  ## Ranking (transitive Abhängige)

  | Rang | Service | Direkt abhängig | Transitiv abhängig | In-Degree |
  ...

  ## Details

  ### <service>
  **Direkte Abhängige:** ...
  **Transitive Abhängige:** ...
  **Upstream (In-Degree):** ...
  ```
  - `generatedAt`-Kommentar in erster Zeile (für Drift-Check filterbar via `grep -v generatedAt`).

- [ ] **C6** — Smoke-Test: `node scripts/build-blast-radius.mjs`
  - `shared-db` auf Rang 1 prüfen.
  - Report hat Inhalt (nicht leer).

## Risiken & Mitigationen

| Risiko | Mitigation |
|--------|-----------|
| IngressRoute-CRD mit Template-Vars (`${PROD_DOMAIN}`) macht YAML-Parse kaputt | Bestehender `try/catch` im YAML-Parser fängt das ab; `${...}`-Werte in `services[].name` werden gefiltert (wie bei `cleanName`) |
| Service-Selector-Matching zu aggressiv (false positives) | Nur exakte Subset-Matches; `traefik` aus Selector-Logik ausschließen |
| Shared-Secret-Kanten erzeugen Kanten zwischen unverwandten Services (z.B. `ghcr-pull-secret`) | Pull-Secrets via `imagePullSecrets` explizit ausschließen; bekannte Infra-Secrets-Allowlist (optional) |
| Blast-Radius BFS auf zyklischem Graphen → Endlos-Schleife | `visited`-Set in BFS; Cycle-Logging |
| `freshness:graph-check` Textdiff schlägt bei Whitespace-Unterschieden an | `diff` mit `-b` (ignore-space-change) oder normalisierter Ausgabe |
