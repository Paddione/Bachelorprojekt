# Admin Live-Architektur-Graph mit Pod-Status-Overlay Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Neue Admin-Seite `/admin/architektur` mit interaktivem D3-Force-SVG-Graphen,
der die statische Architektur (`docs/generated/graph.json`) mit Live-Pod-Status-Overlay
verbindet. Pod-Status-Polling alle 10 s (pausiert bei verborgenem Tab). Klick auf Knoten
öffnet Detail-Panel mit Pod-Status, Warnings und Logs-Link. Beide Brands (BRAND-aware
Namespace-Auflösung). Kein Build-Time-Bundling — graph.json via Admin-API zur Laufzeit.

**Architecture:** Astro SSR-Seite (`/admin/architektur`) + `ArchitekturGraph.svelte`
(Haupt-Orchestrator) + vier Sub-Komponenten (`GraphCanvas`, `GraphNode`, `GraphLegend`,
`NodeDetailPanel`). Neuer API-Endpoint `/api/admin/cluster/graph` liest `graph.json`
und löst `${WEBSITE_NAMESPACE}`/`${WORKSPACE_NAMESPACE}` serverseitig auf.
Pod-Matching: Label `app=<node.id>` → Präfix-Match → "unzugeordnet"-Hinweis.
Status-Overlay: grün/gelb/rot/grau nach Pod-Ready und Restart-Zähler.

**Tech Stack:** Svelte 5 (runes), d3 (bereits im Projekt), Astro SSR, TypeScript,
BATS (Offline-Tests), Vitest (Unit), Node.js `fs.readFile`.
