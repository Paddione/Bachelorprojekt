# Plan: Brett Zuschauer-Modus (T000706)

**Spec:** `docs/superpowers/specs/2026-06-14-t000706-design.md`  
**Branch:** `feature/t000706`

---

## Ziel

Einen dedizierten Zuschauer-Modus für Brett-Boards implementieren: Supervisoren, Ausbilder und
Beobachter erhalten via separatem Zuschauer-Link Zugang zur vollständigen Board-UI, können jedoch
keinerlei Board-Zustände verändern. Alle Board-Events werden live empfangen. Server-seitige
Durchsetzung: fail-closed.

---

## Architektur-Entscheidungen

1. **Token-Typ-Flag** in bestehender `brett_share_tokens`-Tabelle (Spalte `token_type TEXT DEFAULT 'share'`)
   — kein neue Tabelle, gleiche Widerruf-/Ablauf-Logik
2. **`ws._isZuschauer`-Flag** analog zu `ws._isGuest` — separate Semantik (Zuschauer behält
   Identität + volle UI; Gast war anonym mit reduzierter UI)
3. **Eigene URL** `/zuschauer/:token` → `zuschauer.html` (Board-UI mit Zuschauer-Init)
4. **Clientseitige Guards** via `window.__brettIsZuschauer: boolean` — verhindert fehlerhafte
   WS-Sends aus der UI, ohne die Board-Renderpfade zu ändern
5. **Neue Rolle `zuschauer`** im `Role`-Union (types/state.ts) für Teilnehmerliste

  ```typescript
  async function main(): Promise<void> {
    window.__brettIsZuschauer = true;
    const token = location.pathname.split('/').filter(Boolean).at(-1) ?? '';
    const resp = await fetch(`/api/zuschauer/${encodeURIComponent(token)}`);
    if (!resp.ok) { /* Fehlerseite */ return; }
    const { roomToken } = await resp.json();
    const params = new URLSearchParams({ room: roomToken, zuschauer_token: token });
    history.replaceState(null, '', `${location.pathname}?${params.toString()}`);
    // Optional: OIDC-Session prüfen (Zuschauer können eingeloggt sein)
    const board = await import('./board-boot');
    await board.bootBoard();
  }
  main();
  ```
