## Ziel

  (S2): importieren nur Typen, nie die DB-/Route-Schicht.

- **Svelte** (`CockpitSidebar.svelte` / `Cockpit.svelte`) — Batch-Apply statt N+1.
- **Tests:** Vitest. Bestehende Dateien zuerst erweitern statt neue anlegen:
  - `website/src/lib/tickets/__tests__/cockpit-api.test.ts` (217 Z.) — Endpoint-/Parse-/DB-Tests.
  - `website/src/components/admin/SuggestionBar.test.ts` (83 Z.) — falls UI-Verhalten nötig; sonst
    neue `CockpitSidebar.test.ts` für Batch-Apply-Roundtrip-Count.

## Annahmen / Entscheidungen

- **B2-Mechanik:** Statt N Einzel-POSTs ein **ein** Batch-Roundtrip. Es existiert bereits
  `batchMutate()` in `cockpit-db.ts`, aber für `status/priority/parentId` — **nicht** für
  `next_step`. Entscheidung (agentic-first, server-observable): neuer schlanker Endpoint
  `POST /api/admin/cockpit/feature-actions` (Plural) der ein Array `{featureId, action, value}`
  in **einer** Server-Runde verarbeitet (Reuse von `setFeatureAction` in einer Schleife
  server-seitig), und Client ruft ihn genau einmal + einen Refetch. Kleiner, isolierter Endpoint
  statt `batchMutate` zu überladen.
- **B4-Verhalten:** Gutartig leerer/unparsbarer Output → `200 { suggestions: [] }` (deterministisch,
  kein 500). Echter Upstream-Fehler (Netzwerk/Auth/Timeout) bleibt 5xx.
- **B5 fail-closed:** Key-Prüfung **vor** Client-Konstruktion; `'not-required'` wird ersatzlos
  entfernt.

### Task 2 — B3: SSRF — server-seitige Provider-Allowlist
- [ ] **Test first:** Test ergänzen: `provider:'evil'` (nicht in Allowlist) → `400 invalid provider`,

- [ ] **Fix:** Neues pures Modul `website/src/lib/tickets/suggest-providers.ts`:
      `export interface ProviderSpec { id, baseURL?: string, defaultModel, apiKeyEnv?: string }`
      und `export const ALLOWED_PROVIDERS: Record<string, ProviderSpec>` (nur server-bekannte

      Endpoint `baseURL` **nur** aus dieser Tabelle, nie aus dem Client-Body. `suggest.ts`
      verwendet `resolveProvider(body.provider)`; `null` → 400. Pures Modul, nur Typ-Imports (S2).
- [ ] Key-Auflösung (Task 1) zieht jetzt `spec.apiKeyEnv` aus der Allowlist statt hardcodiertem
      Branch. Test grün.
