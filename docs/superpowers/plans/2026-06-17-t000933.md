# Implementierungsplan — T000933 Ticket-Auto-Triage

Spec: `docs/superpowers/specs/2026-06-17-t000933-design.md`
Branch: `feature/t000933`

## Ziel

KI-gestützte Vorklassifizierung neuer, untriagierter Tickets (`status` in
`backlog`/`intake`, `triaged_at IS NULL`) über einen idempotenten Factory-Cron/Poll-
Schritt. DeepSeek schlägt `type`/`priority`/`severity`/`areas`/`component` + einen
`assignee` vor; die Vorschläge landen in `grilling_meta.triage` und `triaged_at`
wird gesetzt. **Kein Auto-Apply** — der Mensch bestätigt im Planungsbüro.

## Architektur / Tech-Stack

- **Poll-Schritt**: `scripts/factory/auto-triage.sh` (Bash), Muster 1:1 von
  `scripts/factory/auto-enqueue.sh` (Pull-Query → pro Ticket KI → Write), wird in
  `scripts/factory/wakeup.sh` direkt nach dem `auto-enqueue`-Loop, vor dem
  Dispatcher-Tick, best-effort (`|| true`) pro Brand aufgerufen.
- **KI-Aufruf**: über den bestehenden Provider-Pfad `route-provider.sh triage flash`
  (DeepSeek). Prompt + JSON-Parse/Validierung über `jq`.
- **Enum-SSOT**: `scripts/factory/triage-enums.json` (versioniert; `areas`,
  `component`-Werte, kuratierte `assignee`-Liste). `type`/`severity` sind die
  bestehenden Ticket-Enums (in der Validierung referenziert).
- **DB**: Spalten `component`, `areas`, `triaged_at`, `grilling_meta` existieren
  bereits (`tickets-db.ts` `ensureSchema`). Idempotente Lücken-Migration
  bereitgestellt für beide Brand-DBs.
- **Review-UI**: Backend `website/src/lib/planning-office.ts` (listOffice +
  neuer apply/discard), API `website/src/pages/api/admin/planungsbuero/[extId].ts`,
  UI `website/src/components/PlanningOffice.svelte`.

## S2/S3/S4-Gates

- **S2 (Zyklen)**: `triage-enums.json` ist reine Datendatei (kein Import).
  `auto-triage.sh` ist Bash (kein TS-Graph). `PlanningOfficeTriage.svelte`
  importiert nur Props/Typen aus `planning-office.ts` — kein Rück-Import in DB-/
  API-Schicht.
- **S3 (Hostnamen)**: keine Brand-Domain-Literale. Cluster-/Namespace-Auflösung
  ausschließlich über `factory_resolve` (wie `auto-enqueue.sh`); keine
  `*.mentolder.de`/`*.korczewski.de`-Strings in Skript oder Snippets.
- **S4 (Orphans)**: `auto-triage.sh` wird von `wakeup.sh` aufgerufen (erreichbar);
  `triage-enums.json` wird von `auto-triage.sh` gelesen; neue BATS-Datei in
  `task test:factory` eingehängt; neue `.svelte`/`.ts` von `PlanningOffice`/Page
  importiert. Kein neues `k3d/*.yaml`. Migration in den Migrations-Ordner + im
  Plan/Doku referenziert.

## Tasks

### Task 1 — Enum-SSOT + KI-Validierungslogik
- [ ] `scripts/factory/triage-enums.json` anlegen: `{ "areas": [...], "components": [...], "assignees": [...] }` (kuratiert, klein).
- [ ] In `auto-triage.sh` (Task 2) eine `validate_triage()` `jq`-Funktion: prüft `type ∈ {bug,feature,task,project}`, `severity ∈ {critical,major,minor,trivial}`, `priority` ∈ erlaubte Werte, `areas ⊆ enums.areas`, `component ∈ enums.components ∨ null`, `assignee_suggested ∈ enums.assignees`. Ungültig → Exit non-zero (fail-closed).
- [ ] BATS: gültiges JSON passes, jedes ungültige Feld failt, Müll-JSON failt.

### Task 2 — `auto-triage.sh` Poll-Skript
- [ ] `scripts/factory/auto-triage.sh` nach Vorbild `auto-enqueue.sh`: `set -euo pipefail`, `source lib.sh`, `--dry-run`/`--help`, `BRAND` required, `FACTORY_DRY_RESOLVE`-Shortcut, `factory_resolve`.
- [ ] Pull-Query: untriagierte `status IN ('backlog','intake') AND triaged_at IS NULL`, `ORDER BY created_at ASC LIMIT ${TRIAGE_BATCH:-5}`.
- [ ] Pro Ticket: Prompt bauen (Titel+Beschreibung+Enums), `route-provider.sh triage flash` → DeepSeek aufrufen, Antwort `jq`-parsen, `validate_triage`. Bei Fehler: `continue` (kein Write).
- [ ] Idempotenter Write: `UPDATE … SET grilling_meta = COALESCE(grilling_meta,'{}') || jsonb_build_object('triage', :sug), triaged_at = now() WHERE external_id=:ext AND triaged_at IS NULL`.
- [ ] OTel-Metrik `factory.triage.count` + `[auto-triage:<brand>]`-Log; `--dry-run` schreibt nicht.
- [ ] Ziel < 200 Zeilen (S1 `.sh`=500, reichlich Reserve).

## Verifikation der Akzeptanzkriterien

- Neues untriagiertes Ticket → nach einem Tick steht `grilling_meta.triage` +
  `triaged_at`; harte Felder unverändert (kein Auto-Apply).
- Zweiter Tick triagiert dasselbe Ticket nicht erneut (Idempotenz über `triaged_at`).
- Müll-/ungültige KI-Antwort → kein Write, Ticket bleibt untriagiert (fail-closed).
- Planungsbüro zeigt Vorschlag; „Übernehmen" setzt echte Spalten, „Verwerfen"
  entfernt nur den Vorschlag.
- `task freshness:check` grün (keine S1–S4-Verletzung, Baseline wächst nicht).
