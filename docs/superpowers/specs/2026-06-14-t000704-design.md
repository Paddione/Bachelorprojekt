## Problemfelder

### P1 — PNG-Auflösung einfach (kein HiDPI)

`canvas.toDataURL()` exportiert in der nativen Canvas-Auflösung. Auf Retina-/HiDPI-Displays
liegt `devicePixelRatio` bei 2–3. Die exportierte PNG hat die physische Bildschirmauflösung
(z.B. 1280×720 CSS px), nicht die physischen Pixel (z.B. 2560×1440). Für Präsentationen
und Protokoll-Anhänge ist 2× besser.

**Lösung:** Offscreen-Canvas 2× + `renderer.render()` auf erhöhtem Pixel-Ratio oder
alternativer Ansatz via `pixelRatio`-Parameter am temporären Three.js-Renderer.
Pragmatischste Option ohne Second-Renderer: `canvas.toDataURL()` gibt die vorhandene
Auflösung aus, aber `preserveDrawingBuffer:true` + temporärer Scale via
`renderer.setPixelRatio(2)` → render → screenshot → restore.

Da `scene.ts` den Renderer als globales Singleton verwaltet, wird der Pixel-Ratio-Ansatz
über eine kurze `snapshot2x()`-Hilfsfunktion in `export.ts` realisiert:

```
snapshot2x(renderer): Promise<string>
  → renderer.setPixelRatio(2)
  → renderer.render(scene, camera)   // einmaliger Off-tick-Render
  → canvas.toDataURL('image/png')
  → renderer.setPixelRatio(original)
  → renderer.render(scene, camera)   // Viewer-Auflösung wiederherstellen
```

`scene.ts` exportiert `getScene()` bereits (`{ renderer, scene, camera, … }`); `export.ts`
importiert den Renderer-Zustand daraus (kein neuer Import-Zyklus, da `scene.ts` bereits
importierbar ist).

### P2 — Kein visuelles Feedback nach Export

Nach `exportPng` / `exportJson` erhält der Nutzer keinen Hinweis, dass der Download
gestartet wurde (Browser-Download-Leiste ist unauffällig). Nach `exportPdf` wird der
Button kurz deaktiviert + `⏳ PDF…`, aber keine Erfolgsmeldung.

**Lösung:** Schlanker Toast (analog `late-join-toast.ts`) — erscheint für 2,5 s oben
rechts mit Text `✓ PNG gespeichert`, `✓ PDF gespeichert`, `✓ JSON gespeichert`.
Fehlerfall: roter Toast `✗ Export fehlgeschlagen`.

### P3 — Dateiname enthält keinen Session-Code

Aktuelle Dateinamen: `brett-2026-06-14.png`. Bei mehreren Sitzungen an einem Tag sind
alle Exporte nicht unterscheidbar.

**Lösung:** Dateiname wird zu `brett-{YYYY-MM-DD}-{sessionCode}.png` erweitert, wenn
`sessionCode` im Cache gesetzt ist. Fallback bleibt `brett-{YYYY-MM-DD}.png`.

### P4 — PDF-Layout bei vielen Figuren/Linien überläuft

Die aktuelle Figurenliste in `exportPdf` zeigt maximal 8 Zeilen pro Spalte. Bei
>32 Figuren überläuft die Liste aus dem Seitenrahmen (jsPDF bricht nicht automatisch
auf neue Seiten um).

**Lösung:** Seitenumbruch via `doc.addPage()` wenn die Y-Position > 185 mm (A4-Landscape
Seitenhöhe 210 mm, Bottom-Margin 25 mm). Neue Seite startet bei Y = 20 mm.
