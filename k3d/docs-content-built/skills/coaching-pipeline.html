Coaching Pipeline

End-to-End Workflow für Ingestion, Klassifizierung und Review von Coaching-Inhalten aus PDF/EPUB in die Live-Knowledgebase.

Auslöser: Ingestion neuer Bücher, Klassifizierung von Chunks oder Draft-Review

Pipeline Übersicht

Source file (PDF/EPUB)
    ↓  task coaching:ingest
Chunks stored in pgvector (status: UNCLASSIFIED)
    ↓  task coaching:classify
Chunks classified by LLM (status: DRAFT)
    ↓  /admin/knowledge/drafts review
Chunks promoted to PUBLISHED
    ↓  available to coaching assistant
1 Quelldatei ingestieren PDF/EPUB → pgvector 
task coaching:ingest -- <file> <slug> [--title="..."] [--author="..."]

Splittet die Quelle in Chunks, generiert Embeddings und speichert sie mit Status UNCLASSIFIED.

Check:

task workspace:psql ENV=mentolder -- website <<'SQL'
SELECT slug, title, chunk_count FROM coaching.books ORDER BY created_at DESC LIMIT 5;
SQL
2 Chunks klassifizieren KI-Kategorisierung 
task coaching:classify -- --slug=<slug>
task coaching:classify -- --all

Nutzt LLM (Ollama/Anthropic) zur Zuweisung von Themen und Confidence-Scores. Status wechselt auf DRAFT.

ℹ️
Embedding Modell: bge-m3 nutzt TEI (llm-gateway-embed:8081). Voyage AI wird direkt angesprochen. Modelle dürfen nie gemischt werden.
3 Review & Freischaltung /admin/knowledge/drafts

Prüfe die Chunks im Admin-Interface und schalte sie mit Approve frei.

Bulk-Approve via SQL (Vorsicht):

UPDATE coaching.chunks SET status = 'PUBLISHED', reviewed_at = now()
WHERE book_id = (SELECT id FROM coaching.books WHERE slug = '<slug>')
  AND status = 'DRAFT';
4 Verfügbarkeit prüfen Coaching Assistant Test

Verifiziere, dass Chunks im Portal abrufbar sind:

SELECT status, COUNT(*) FROM coaching.chunks GROUP BY status;

Erwartet: Mischung aus PUBLISHED, null UNCLASSIFIED.
