Deployment Assist

Geführte Bereitstellung der Plattform: Assessment von Tools, Credentials und Clustern mit anschließender sequentieller Ausführung aller Setup-Tasks.

Auslöser: Erst-Deployment, Recovery nach Cluster-Reset oder Diagnose von Deploy-Fehlern
0 Environment Selection Ziel-Cluster wählen

Wähle zwischen dev (k3d), mentolder (Prod), korczewski (Prod) oder all-prods.

⚠️
Context-Check: Prüfe vor dem Deployment immer, ob der aktive kubectl Context zum Ziel-Cluster passt.
1 Prerequisite Check Tools & Config

Prüfe Vorhandensein von: docker, kubectl, task, flux, kubeseal.

task env:validate ENV=<env>
flux check --context <ctx>
2 Gap Analysis Was fehlt?

Erstelle eine Übersicht über den Status von:

  • Sealed Secrets: Vorhanden und entschlüsselbar?
  • Flux: GitRepository und Kustomizations Ready?
  • Pods: Core-Services Running?
  • Post-Setup: Nextcloud konfiguriert? Talk HPB aktiv?
3 Execution Tasks in Reihenfolge

Führe die benötigten Tasks sequentiell aus. Beispiel für ein frisches Dev-Setup:

1. task cluster:create
2. task workspace:up

Beispiel für Prod Recovery:

1. task sealed-secrets:install
2. task env:fetch-cert
3. task env:seal
4. task workspace:deploy
