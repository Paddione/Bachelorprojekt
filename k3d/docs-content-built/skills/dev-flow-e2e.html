Dev Flow E2E

Automatisierte Browser-Tests mit Playwright: Live-Erkundung, Test-Spec Erstellung und Verifizierung der implementierten Features gegen die Produktionsumgebung.

Auslöser: Abschluss von dev-flow-execute (Merge & Deploy)
1 Live-Erkundung MCP Playwright Browser

Nutze die MCP Browser-Tools, um das implementierte Feature live zu testen und Interaktionspunkte zu identifizieren:

mcp__browser_navigate({ url: "https://web.mentolder.de/<path>" })
mcp__browser_snapshot({})
mcp__browser_network_requests({})
2 Test-Spec schreiben tests/e2e/specs/fa-NN-*.spec.ts

Erstelle einen neuen Test-Spec (Nummerierung fortlaufend aus tests/e2e/specs/):

import { test, expect } from '@playwright/test';

test('FA-NN: Feature Test', async ({ page }) => {
    await page.goto('/');
    await expect(page).toHaveTitle(/Workspace/);
});
3 Verifizierung Run & Inventory Update

Führe den Test lokal gegen die Live-URL aus:

WEBSITE_URL=https://web.mentolder.de npx playwright test --project website

Aktualisiere danach das Test-Inventory:

task test:inventory
