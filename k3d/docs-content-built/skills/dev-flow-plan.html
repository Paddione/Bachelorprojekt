Dev Flow Plan

Einstiegspunkt für alle Änderungen: Analyse der Anforderung, Wahl des Arbeits-Pfades (Feature, Fix oder Chore) und Erstellung technischer Spezifikationen.

Auslöser: Jede neue Arbeitsanfrage (Feature-Wunsch, Bug-Meldung, Maintenance)

Pfade & Entscheidungsmatrix

PfadWann nutzen?Ergebnis
featureNeues Verhalten, UI-Sektionen, API-EndpunkteWorktree → Spec → Plan → Ticket
fixEtwas ist kaputt oder weicht vom Soll abRed-Test → Worktree → Fix-Plan
choreRefactoring, Doku, Bumps, Config-TweaksDirekte Ausführung & PR
1 Vorbereitung Git & Worktree Audit

Synchronisiere main und bereinige verwaiste Worktrees:

git pull --rebase origin main
git worktree list
2 Design & Brainstorming Visual Companion Server

Starte bei Features den Brainstorming-Tunnel für visuelle Kollaboration:

task brainstorm:publish -- 3000

Die URL https://brainstorm.mentolder.de ermöglicht Echtzeit-Design-Entscheidungen.

3 Spezifikation & Ticket Postgres-Archivierung

Schreibe Spec und Plan, lege danach das Ticket in der Plattform-DB an:

task tickets:create --title "Plan: <slug>" --type task

Der fertige Plan wird im Frontmatter mit der ticket_id verknüpft.
