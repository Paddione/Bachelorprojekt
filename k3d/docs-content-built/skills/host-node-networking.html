Netplan host-node-networking

Host, Firewalls & VPN Mesh

Host node provisioning, WireGuard mesh networking configuration ("netplan"), host firewall (UFW) rules, LiveKit WebRTC setups, and WSL OpenClaw gateway operations.

Auslöser: Network config change, node provisioning required, LiveKit ICE issues, or WSL Ollama API changes.
ℹ️
VPN Mesh ("Netplan"): Physical nodes and workstations form a unified WireGuard VPN network overlay, securing all backend API and signaling traffic.
1 Hetzner Node Provisioning Cloud-Init or Rescue Mode Reset

Determine server details (New vs. Reset) and role (Control Plane or Worker). Retrieve active tokens:

ssh patrick@<CP_IP> "sudo cat /var/lib/rancher/k3s/server/node-token"

Inject user data cloud-config or boot via Rescue Mode and run the setup script.

2 Host Firewall Setup (UFW) Rules & Port Mappings

Open required ports directly on nodes (SSH, Traefik, VPN, coturn, LiveKit RTC):

ssh patrick@<node-ip> "sudo ufw allow 7880/tcp && sudo ufw reload"
3 LiveKit WebRTC Stack DNS Pinning & Affinity

LiveKit is pinned to `gekko-hetzner-3`. Ensure DNS records point directly to its public IP:

task livekit:dns-pin ENV=mentolder APPLY=true
4 WSL OpenClaw Gateway Local Ollama Operations

Deploy the local gateway on WSL to route chat calls over the WireGuard VPN to the GPU worker:

task openclaw:start

Troubleshooting

SymptomLösung
VPN Handshake failsRe-derive public key and compare with registry configuration.
Flannel traffic blockedEnsure UDP 8472 and VPN port 51820 are allowed in host UFW rules.
LiveKit ICE failsVerify DNS matches the node hosting the LiveKit container.
OpenClaw 503Confirm Ollama is running on the GPU host (10.10.0.3).
