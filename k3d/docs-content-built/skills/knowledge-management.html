AI knowledge-management

Knowledge Ingestion & Indexing

Ingesting coaching resources (PDFs, EPUBs), running the classifier LLM pipeline, re-indexing workspace collections, managing web source crawlers, and ensuring vector space model isolation.

Auslöser: Ingesting books, indexing PRs/docs/bugs, or resolving vector schema mismatches.
ℹ️
Model Isolation: bge-m3 (local TEI container) and voyage-multilingual-2 (Voyage AI cloud) are completely incompatible vector spaces. Never mix them in the same query.
1 Coaching Ingest Split & Store Chunks

Load PDFs or EPUB files into pgvector. Chunks start with status = 'UNCLASSIFIED':

task coaching:ingest -- /path/to/book.pdf book-slug
2 LLM Classification Categorization Workflow

Classify chunks using chat LLM (in-cluster router or Anthropic API fallback) into DRAFT status:

task coaching:classify -- --slug=book-slug
3 Admin Review Approve Drafts

Open /admin/knowledge/drafts to review draft categories. Once approved, chunks are promoted to PUBLISHED and ready for searches.

4 General Reindexing PRs, Docs & Tickets

Trigger re-indexing for other workspace knowledge bases:

task knowledge:reindex ENV=mentolder SOURCE=prs

Sources: prs, markdown (docs), bugs (tickets), or all.
