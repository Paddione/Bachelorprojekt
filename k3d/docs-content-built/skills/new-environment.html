New Environment Bring-up

Initialisierung eines neuen Kubernetes-Clusters: Von der Konfigurations-Erstellung über Sealed Secrets und Cert-Manager bis zum finalen Workspace-Deployment.

Auslöser: Aufbau eines neuen Standorts oder vollständiger Cluster-Reset
🚫
Zwingende Reihenfolge: Das Überspringen von Schritten (z.B. Deploy vor Seal) führt dazu, dass Platzhalter-Secrets in Produktion landen oder Ressourcen fehlschlagen.
1 Config-Scaffolding Task env:init

Erstelle das Environment-File basierend auf dem Schema:

task env:init ENV=<new-env>
task env:validate ENV=<new-env>
2 Sealed Secrets Setup Controller & Cert Fetch

Installiere den Controller und hole das Zertifikat des neuen Clusters:

task sealed-secrets:install ENV=<env>
task env:fetch-cert ENV=<env>
3 Secrets & Certificates Generate, Seal & Sync

Generiere und verschlüssele die Secrets, installiere Cert-Manager:

task env:generate ENV=<env>
task env:seal ENV=<env>
task cert:install ENV=<env>
4 Deployment & Post-Setup Bring-up des Workspace

Rolle den Workspace aus und konfiguriere die Dienste:

task workspace:deploy ENV=<env>
task workspace:post-setup ENV=<env>
task workspace:talk-setup ENV=<env>

Deployment Checkliste

  • 1
    Office Deployment: Separater Task für Collabora.
  • 2
    Admin Users: Initiales Passwort-Seeding.
  • 3
    Keycloak Sync: Realm-Konfiguration importieren.
