Secret Rotation

Sichere und koordinierte Erneuerung von Credentials: DB-Passwörter, API-Keys, Sealed Secrets Keypairs und Service-Tokens über alle Cluster hinweg.

Auslöser: Kompromittierung, Passwort-Drift nach Reset oder periodische Rotation
🚫
Kritische Reihenfolge: Starte niemals task workspace:deploy, bevor task env:seal erfolgreich abgeschlossen wurde. Andernfalls überschreibt das Deployment Produktions-Secrets mit Entwicklungs-Platzhaltern.

Rotations-Typen

TypSzenarioWichtigster Befehl
ADB Password Drifttask workspace:sync-db-passwords
BNeue Secrets / Rotationtask env:generate & env:seal
CCluster / Controller Resettask env:fetch-cert vor dem Sealing
DClaude Code Tokenstask claude-code:rotate-tokens
1 Vorbereitung Cert Fetch (nur bei Reset)

Hole das aktuelle Sealing-Zertifikat des Clusters ab:

task env:fetch-cert ENV=mentolder
2 Generation & Sealing env:generate & env:seal

Erzeuge neue Klartext-Secrets (gitignored) und verschlüssele sie:

task env:generate ENV=mentolder
task env:seal ENV=mentolder
3 Aktivierung Sync & Restart

Wende die verschlüsselten Secrets an und starte betroffene Pods neu:

task secrets:sync
task workspace:restart ENV=mentolder -- <service>
4 DB Sync Nur Typ A

Gleiche DB-Rollen-Passwörter an die neuen Secrets an:

task workspace:sync-db-passwords ENV=mentolder

Cluster-Checkliste

ℹ️
Jeder Cluster hat seinen eigenen Controller und sein eigenes Zertifikat. Secrets müssen separat pro Environment verschlüsselt und angewendet werden. Ein mentolder-SealedSecret funktioniert nicht auf korczewski.
