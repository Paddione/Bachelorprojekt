Using Git Worktrees

Verwaltung isolierter Arbeitsumgebungen: Korrektes Setup von Submodulen, Symlinking von Secrets und Branch-Renaming für den Bachelorprojekt Git-Flow.

Auslöser: Start einer neuen Feature- oder Fix-Implementierung
❗
Pull First: Aktualisiere immer origin/main, bevor du einen neuen Worktree anlegst, um Merge-Konflikte zu minimieren.
1 Worktree anlegen Isolation schaffen

Erstelle einen neuen Worktree für den Branch:

git worktree add .claude/worktrees/<name> feature/<slug>
2 Submodule Init T000387

Da git worktree add keine Submodule initialisiert, muss dies manuell nachgeholt werden (für BATS Tests):

git submodule update --init --recursive
3 Secret Symlinking T000383

Verknüpfe die Secrets des Haupt-Repos, damit Tasks im Worktree funktionieren:

ln -sfn /home/patrick/Bachelorprojekt/environments/.secrets \
       environments/.secrets
4 Branch Renaming T000381

Falls Tools wie EnterWorktree Slashes im Branch-Namen kodiert haben (+), benenne den Branch um:

git branch -m feature/my-task

Automatisierung

In diesem Repository feuert ein PostToolUse Hook (Matcher: EnterWorktree), der die Schritte 2 und 3 automatisch übernimmt. Überprüfe die Statusmeldung: "Initializing submodules and symlinking secrets..."
