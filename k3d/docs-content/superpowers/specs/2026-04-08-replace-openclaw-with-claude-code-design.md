# Replace OpenClaw with Claude Code — Hybrid Approach

**Date:** 2026-04-08
**Status:** Draft
**Author:** Patrick + Claude

## Goal

Replace the self-hosted OpenClaw AI assistant (Open-WebUI + Anthropic API) with Claude Code as the interface for managing the mentolder.de workspace. Two separate Claude Code environments provide role-based access to the existing MCP server infrastructure.

## Non-Goals

- No client-facing chat UI (coaching clients don't interact with AI)
- No GPU worker scheduling (future scope, separate spec)
- No changes to MCP server implementations themselves

## Components

### 1. ForwardAuth Proxy (mcp-auth-proxy)

A minimal nginx container that validates bearer tokens on every MCP request.

**How it works:**
1. Traefik receives request to `mcp-mentolder.de/{service}/mcp`
2. ForwardAuth middleware calls `mcp-auth-proxy` with the original headers
3. nginx extracts the `Authorization: Bearer <token>` header
4. Checks token against `CLUSTER_TOKEN` and `BUSINESS_TOKEN` env vars (from `mcp-tokens` Secret)
5. If valid, returns `200` with `X-MCP-Role: cluster|business` header
6. If invalid, returns `401`

**Role-based path enforcement:**
The nginx config contains a path-to-role mapping. After token validation, it checks whether the token's role is authorized for the requested path.

| Path Prefix | Required Role |
|-------------|--------------|
| `/kubernetes` | cluster |
| `/postgres` | cluster |
| `/keycloak` | cluster |
| `/browser` | cluster |
| `/github` | cluster |
| `/mattermost` | cluster, business |
| `/nextcloud` | cluster, business |
| `/invoiceninja` | cluster, business |
| `/stripe` | cluster, business |

If a business token tries to access `/kubernetes`, the proxy returns `403`.

**File:** `deploy/mcp/mcp-auth-proxy.yaml`
- Deployment: 1 replica, nginx:alpine image
- ConfigMap: nginx.conf with token validation + path authorization logic
- Service: ClusterIP on port 80

### 2. MCP Tokens Secret

**File:** `deploy/mcp/mcp-tokens.yaml`

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: mcp-tokens
type: Opaque
stringData:
  CLUSTER_TOKEN: "<generated-at-deploy-time>"
  BUSINESS_TOKEN: "<generated-at-deploy-time>"
```

Tokens are 64-character random hex strings generated by the Taskfile setup command. Production values go in `prod/secrets.yaml` (never committed).

### 3. MCP Status Page

Replaces OpenClaw WebUI at `ai.mentolder.de` with a read-only health dashboard.

**File:** `deploy/mcp/mcp-status.yaml`

**Components:**
- nginx deployment serving a static HTML page
- Sidecar container running a health-check loop (curl each MCP endpoint every 30s, writes JSON to shared volume)
- HTML page auto-refreshes, reads the JSON, renders a status grid

**Health check targets:**
| Server | Endpoint |
|--------|----------|
| kubernetes | `http://openclaw-mcp-core:8080/mcp` |
| postgres | `http://openclaw-mcp-core:3001/mcp` |
| mattermost | `http://openclaw-mcp-core:8000/mcp` |
| nextcloud | `http://openclaw-mcp-apps:8000/health/ready` |
| invoiceninja | `http://openclaw-mcp-apps:8080/mcp` |
| keycloak | `http://openclaw-mcp-auth:8080/q/health` |
| browser | `http://openclaw-mcp-browser:3000/mcp` |
| github | `http://mcp-github:3002/mcp` |
| stripe | `http://mcp-stripe:3003/mcp` |

**Display:** Two groups — "Cluster Management" and "Business" — each showing server name, status badge (green/red), and last-checked timestamp. No authentication required (no sensitive data exposed).

- Keep all existing path-based routes

New middleware chain:
```
mcp-chain:
  - mcp-forwardauth    (replaces mcp-ipallow)
  - mcp-strip-service-prefix
  - security-headers
  - compression
```

ForwardAuth middleware definition:
```yaml
apiVersion: traefik.io/v1alpha1
kind: Middleware
metadata:
  name: mcp-forwardauth
spec:
  forwardAuth:
    address: http://mcp-auth-proxy:80/auth
    authResponseHeaders:
      - X-MCP-Role
```

### 6. Taskfile Commands

**New task: `claude-code:setup`**
```bash
task claude-code:setup -- cluster|business
```
1. Reads token from `mcp-tokens` Secret via kubectl
2. Reads `PROD_DOMAIN` from `.env`
3. Runs `envsubst` on the template file
4. Writes to `~/.claude/settings.json`
5. Prints confirmation with the configured MCP servers

**New task: `claude-code:rotate-tokens`**
```bash
task claude-code:rotate-tokens
```
1. Generates new 64-char hex tokens
2. Updates `mcp-tokens` Secret
3. Restarts `mcp-auth-proxy`
4. Prints reminder to re-run `claude-code:setup` on each machine

## Security Considerations

- **Tokens over HTTPS only:** Traefik terminates TLS; tokens never traverse the network in plaintext.
- **Token rotation:** `claude-code:rotate-tokens` task generates new tokens and restarts the auth proxy. Users must re-run `claude-code:setup` after rotation.
- **No IP restriction removed without replacement:** The IP allowlist is replaced by bearer token auth, not just removed. Both provide access control; tokens are more flexible for remote access.
- **Role enforcement:** Business tokens cannot reach cluster-management endpoints. Enforced at the ForwardAuth proxy level, not just by config convention.
- **Status page exposes no secrets:** Only shows server names and up/down status. No tokens, credentials, or internal URLs visible.

## Testing

- Verify ForwardAuth rejects requests without a token (expect `401`)
- Verify ForwardAuth rejects invalid tokens (expect `401`)
- Verify business token cannot access `/kubernetes` (expect `403`)
- Verify cluster token can access all endpoints (expect `200`)
- Verify Claude Code can connect to each MCP server via the config template
- Verify status page shows accurate health for all MCP servers
- Verify token rotation works end-to-end
