## ADDED Requirements

### Requirement: Automated accessibility scanning tooling

The system SHALL provide an automated axe-core accessibility scan of the core
routes for both brands (mentolder, korczewski), runnable both via a manual
task wrapper and as part of the nightly E2E CI run, gating on `critical` and
`serious` impact violations.

#### Scenario: Manual pre-merge a11y scan via task wrapper

- **GIVEN** a developer wants to check for accessibility regressions before merging
- **WHEN** they run `task a11y:axe ENV=mentolder` (or `ENV=korczewski`)
- **THEN** Playwright runs `tests/e2e/specs/a11y-axe.spec.ts` with `@axe-core/playwright`
  against the core routes of that brand and reports any `critical`/`serious`
  violations found

#### Scenario: Nightly CI accessibility regression check

- **GIVEN** the nightly `e2e.yml` workflow runs `npx playwright test` without a
  project filter
- **WHEN** the `website` project executes
- **THEN** `a11y-axe.spec.ts` (matched via `playwright.config.ts` `testMatch`)
  scans the core routes of both brands and fails the run if any route has
  `critical`/`serious` violations
