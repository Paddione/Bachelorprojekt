## ADDED Requirements

### Requirement: pre-commit freshness auto-stage covers all regen-produced files

The `.githooks/pre-commit` hook SHALL auto-stage every file regenerated by
`task freshness:regenerate`, not only a hardcoded subset. The auto-stage
manifest SHALL be a superset of the `FILES` variable used by
`task freshness:check` in `Taskfile.yml`, so that the post-commit state is
fresh by definition and no amend+re-push is required to clear a CI
freshness-check failure.

#### Scenario: openspec-status.json is auto-staged after a regen that touches it

- **GIVEN** `task openspec:status-map` writes a new value to
  `website/src/data/openspec-status.json` during a pre-commit run
- **WHEN** the pre-commit hook iterates its `_FRESHNESS_FILES` list and
  runs `git add` on the changed entries
- **THEN** `website/src/data/openspec-status.json` is in the list and is
  staged into the commit
- **AND** the freshness check in CI does not see a drift for this file

#### Scenario: loc-budget.json is auto-staged after a regen that touches it

- **GIVEN** `task loc:update-baseline` writes a new value to
  `docs/code-quality/loc-budget.json` during a pre-commit run
- **WHEN** the pre-commit hook iterates its `_FRESHNESS_FILES` list and
  runs `git add` on the changed entries
- **THEN** `docs/code-quality/loc-budget.json` is in the list and is
  staged into the commit
- **AND** the freshness check in CI does not see a drift for this file

#### Scenario: pre-commit auto-stage list is a superset of freshness:check FILES *(BATS)*

- **GIVEN** the `FILES` variable in `Taskfile.yml` under
  `freshness:check` and the `_FRESHNESS_FILES` array in
  `.githooks/pre-commit`
- **WHEN** both lists are extracted and compared
- **THEN** every path in the `freshness:check` `FILES` variable is also
  present in the pre-commit `_FRESHNESS_FILES` array
- **AND** the test fails with a list of missing entries when drift is
  detected
