## ADDED Requirements

### Requirement: Secret Rotation After Transcript Exposure

The system SHALL automatically rotate all secrets when a transcript exposure event is detected or initiated.

#### Scenario: Rotating Secrets for All Environments

- **GIVEN** an admin initiates a transcript exposure cleanup
- **WHEN** the rotation process is triggered via CLI command
- **THEN** all environment-specific secrets must be regenerated and re-sealed:
  - `env-generate.sh` creates new secrets in environments/.secrets/<env>.yaml
  - `env-seal.sh` encrypts them as SealedSecret for each environment (dev, mentolder, korczewski)
  - The deploy pipeline picks up the new sealed secrets and applies them to Kubernetes

#### Scenario: Verification of Rotation Success

- **GIVEN** secrets have been rotated
- **WHEN** running `task test:changed`
- **THEN** all freshness artifacts (test-inventory.json, route-manifest.json) are updated and committed
